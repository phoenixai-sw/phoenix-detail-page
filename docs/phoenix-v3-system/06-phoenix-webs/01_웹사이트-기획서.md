# phoenix webs 웹사이트 기획서

## 1. 정의

`phoenix webs`는 웹사이트/랜딩페이지 HTML 파일 생성 웹이다.

사용자가 서비스명, 목적, 섹션 구성, 이미지 자료, 톤앤매너를 입력하면 간단한 랜딩페이지 HTML 결과물을 생성한다.

## 2. 주 컨트롤 봇

```text
Design Bot
```

Design Bot은 웹디자인, 랜딩페이지 구조, 브랜드 비주얼, 섹션 흐름을 다루기 때문에 `phoenix webs`를 컨트롤하는 주 봇으로 적합하다.

## 3. 주요 유저

1. 수강생 유저
2. 소상공인
3. 강사
4. 이벤트 운영자
5. 스타트업 초기 팀
6. 랜딩페이지를 빠르게 확인하려는 기획자

## 4. 핵심 기능

1. 웹사이트 목적 입력
2. 브랜드명/서비스명 입력
3. 랜딩페이지 유형 선택
4. 섹션 구성 선택
5. 이미지/로고 업로드
6. CTA 문구 입력
7. HTML 생성
8. 브라우저 미리보기
9. HTML/CSS/이미지 ZIP 다운로드
10. V3 이후 Design Bot 자동 조작

## 5. 추천 메뉴

1. 강의 모집 랜딩페이지
2. 세미나/이벤트 신청 페이지
3. 서비스 소개 페이지
4. 제품 소개 페이지
5. 개인 브랜딩 페이지
6. 간단한 사전예약 페이지

## 6. 결과물

1. `index.html`
2. `style.css`
3. 이미지 assets 폴더
4. ZIP 파일
5. 미리보기 URL 또는 로컬 미리보기

## 7. V3 자동화 흐름

```text
Genesis Bot 또는 마스터
→ Design Bot에게 랜딩페이지 생성 요청
→ Design Bot이 phoenix webs 접속
→ 브랜드명/목적/섹션/CTA 입력
→ 이미지 업로드
→ HTML 생성
→ ZIP 다운로드
→ Telegram 또는 Cloud로 전달
```

## 8. 구축 우선순위

`phoenix webs`는 `phoenix pages`, `phoenix slides` 다음에 구축하는 것이 좋다.

이유:

1. Design Bot의 기존 성격과 잘 맞는다.
2. HTML 랜딩페이지는 결과 확인이 빠르다.
3. 수강생 유저가 바로 사용할 수 있다.
4. V3에서 “봇이 웹을 만들어 주는 웹”이라는 상징성이 크다.
