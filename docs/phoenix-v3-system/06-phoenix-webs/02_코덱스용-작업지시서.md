# phoenix webs 코덱스용 작업지시서

## 1. 작업 목표

`phoenix webs`를 Design Bot이 컨트롤할 수 있는 웹사이트/랜딩페이지 HTML 생성 웹으로 구축한다.

## 2. 핵심 화면

1. 웹사이트 목적 입력
2. 브랜드/서비스 정보 입력
3. 랜딩페이지 유형 선택
4. 섹션 구성 선택
5. 이미지/로고 업로드
6. HTML 생성 진행
7. 미리보기
8. ZIP 다운로드

## 3. 자동화용 식별자

```text
data-testid="phoenix-webs-purpose"
data-testid="phoenix-webs-brand-name"
data-testid="phoenix-webs-page-type"
data-testid="phoenix-webs-section-list"
data-testid="phoenix-webs-upload"
data-testid="phoenix-webs-cta"
data-testid="phoenix-webs-generate"
data-testid="phoenix-webs-preview"
data-testid="phoenix-webs-download-zip"
```

## 4. API 후보

```text
src/app/api/webs/generate/route.ts
src/app/api/webs/preview/route.ts
src/app/api/webs/export-zip/route.ts
```

## 5. 데이터 구조 후보

```text
web_projects
web_sections
web_assets
web_exports
```

## 6. 생성 결과 기준

1. HTML은 단일 파일로도 열릴 수 있어야 한다.
2. ZIP에는 `index.html`, `style.css`, `assets`가 포함된다.
3. 모바일 반응형을 기본으로 한다.
4. CTA 버튼이 명확해야 한다.
5. 텍스트가 영역 밖으로 넘치지 않아야 한다.

## 7. 금지 사항

1. 처음부터 복잡한 CMS를 만들지 않는다.
2. 실결제/회원가입 기능을 자동 생성하지 않는다.
3. 외부 스크립트를 무분별하게 넣지 않는다.
4. 봇 자동화를 화면 텍스트에만 의존하지 않는다.
5. 생성된 HTML에 API 키나 민감정보를 넣지 않는다.

## 8. 1차 성공 기준

1. 입력값 기반 랜딩페이지 생성
2. 미리보기 가능
3. ZIP 다운로드 가능
4. Design Bot 자동 조작 가능
