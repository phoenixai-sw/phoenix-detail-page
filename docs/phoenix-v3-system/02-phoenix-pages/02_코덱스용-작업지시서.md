# phoenix pages 코덱스용 작업지시서

## 1. 작업 목표

현재 `phoenix detail page`를 `phoenix pages`로 정리하고, Design Bot이 안정적으로 컨트롤할 수 있는 상세 페이지 생성 웹으로 발전시킨다.

## 2. 주요 수정 대상

```text
src/components/redesign-wizard.tsx
src/app/api/generate/route.ts
src/app/api/edit-section/route.ts
src/app/api/ai-comment/route.ts
src/app/page.tsx
src/app/layout.tsx
docs/upgrade-roadmap
```

## 3. 필수 작업

### 3.1 명칭 정리

화면과 문서에서 아래 기준을 사용한다.

```text
제품명: phoenix pages
설명: 상세 페이지 생성 웹
```

기존 `phoenix detail page`는 저장소/프로젝트명으로 남을 수 있지만, 유저가 보는 명칭은 `phoenix pages`로 정리한다.

### 3.2 봇 자동화용 식별자 추가

Design Bot이 안정적으로 조작할 수 있도록 주요 요소에 `data-testid`를 추가한다.

필수 예시:

```text
data-testid="phoenix-pages-upload"
data-testid="phoenix-pages-openai-key"
data-testid="phoenix-pages-google-key"
data-testid="phoenix-pages-channel"
data-testid="phoenix-pages-page-count"
data-testid="phoenix-pages-aspect-ratio"
data-testid="phoenix-pages-prompt"
data-testid="phoenix-pages-generate"
data-testid="phoenix-pages-result-gallery"
data-testid="phoenix-pages-download-zip"
data-testid="phoenix-pages-cloud-save"
```

### 3.3 결과 다운로드 안정화

봇 자동화에서는 ZIP 다운로드가 가장 중요하다.

필수 조건:

1. 결과가 없으면 버튼 비활성화
2. ZIP 파일명에 프로젝트명과 날짜 포함
3. 다운로드 중 상태 표시
4. 다운로드 실패 메시지 제공

### 3.4 Cloud 저장 대비

V1에서 아래 구조와 연결할 수 있게 설계한다.

```text
cloud_projects
cloud_project_sections
cloud_project_images
```

### 3.5 크레딧 대비

V1.5에서 생성/편집 액션에 대해 사용량 로그를 남길 수 있게 한다.

```text
action = page_generate
action = page_edit
action = ai_comment
```

## 4. 금지 사항

1. 현재 생성 기능을 깨지 않는다.
2. 비로그인 유저의 로컬 다운로드를 막지 않는다.
3. 봇 자동화 때문에 화면을 너무 기계적으로 만들지 않는다.
4. 유저 API 키를 DB에 저장하지 않는다.
5. `redesign-wizard.tsx`를 한 번에 대규모 리팩터링하지 않는다.

## 5. 검증

```bash
npm run build
```

수동 확인:

1. 1장 생성
2. 4장 생성
3. 8장 생성
4. 페이지 추가 생성
5. 개별 편집
6. ZIP 다운로드
7. Design Bot 자동화 식별자 존재 확인
