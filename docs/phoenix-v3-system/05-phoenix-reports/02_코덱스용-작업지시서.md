# phoenix reports 코덱스용 작업지시서

## 1. 작업 목표

`phoenix reports`를 Power Bot이 컨트롤할 수 있는 리서치/보고서 생성 웹으로 구축한다.

## 2. 핵심 화면

1. 보고서 주제 입력
2. 보고서 유형 선택
3. 자료 업로드
4. 목차 미리보기
5. 본문 생성
6. 근거 확인
7. 다운로드

## 3. 자동화용 식별자

```text
data-testid="phoenix-reports-topic"
data-testid="phoenix-reports-type"
data-testid="phoenix-reports-upload"
data-testid="phoenix-reports-research-range"
data-testid="phoenix-reports-generate-outline"
data-testid="phoenix-reports-generate-body"
data-testid="phoenix-reports-preview"
data-testid="phoenix-reports-download-pdf"
data-testid="phoenix-reports-download-docx"
```

## 4. API 후보

```text
src/app/api/reports/outline/route.ts
src/app/api/reports/generate/route.ts
src/app/api/reports/export-pdf/route.ts
src/app/api/reports/export-docx/route.ts
```

## 5. 근거 정책

보고서에는 다음 정보를 가능한 한 포함한다.

1. 출처명
2. 링크
3. 발행일
4. 확인일
5. 요약

## 6. 금지 사항

1. 최신성이 필요한 내용을 검색 없이 단정하지 않는다.
2. 출처 없는 수치를 보고서에 넣지 않는다.
3. 유저가 업로드한 내부자료를 외부에 노출하지 않는다.
4. Power Bot이 실패한 리서치를 성공으로 표시하지 않는다.

## 7. 1차 성공 기준

1. 보고서 목차 생성
2. 본문 초안 생성
3. PDF 다운로드
4. Power Bot 자동 조작 가능
