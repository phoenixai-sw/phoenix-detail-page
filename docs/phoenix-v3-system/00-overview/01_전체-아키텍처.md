# Phoenix V3 전체 아키텍처

## 1. 전체 개념

Phoenix V3는 봇과 웹사이트가 서로 연결되는 생성형 업무 시스템이다.

기존 방식:

```text
사람이 웹사이트에 접속
→ 파일 업로드
→ 버튼 클릭
→ 결과 다운로드
```

V3 방식:

```text
마스터가 Genesis Bot에게 요청
→ Genesis Bot이 작업을 정리
→ phoenix command에 작업 등록
→ 전문 봇에게 작업 배분
→ 전문 봇이 agent web을 컨트롤
→ 결과 생성
→ Telegram / Cloud / Dashboard로 전달
```

## 2. 지휘 계층

### 2.1 phoenix command

Phoenix V3의 최상위 지휘 웹이다.

주요 기능:

1. 작업 요청 등록
2. 봇별 작업 상태 확인
3. 결과물 모음 보기
4. 실패 작업 재시도
5. 승인/크레딧 사용 확인
6. 슈퍼 관리자 모드
7. 유저별 관리자 모드

## 5. 데이터 계층

V1 이후에는 Supabase를 기준으로 한다.

```text
Supabase Auth = Google 로그인
Supabase Database = 유저, 프로젝트, 작업, 결과 메타데이터
Supabase Storage = 이미지, PPT, 영상, 보고서, 상담 문서 파일 저장
```

V1.5 이후에는 크레딧 저장소가 추가된다.

```text
credit_accounts
credit_ledger
payment_orders
usage_logs
```

## 6. 공통 작업 상태

모든 agent web과 봇은 같은 작업 상태 언어를 사용한다.

```text
requested
accepted
waiting_for_input
waiting_for_approval
running
generating
uploading
downloading
sending
completed
failed
cancelled
needs_master_review
```

## 7. V3 성공 기준

1. 마스터가 Genesis Bot에게 자연어로 요청한다.
2. Genesis Bot이 작업을 정리한다.
3. phoenix command에 작업이 등록된다.
4. 전문 봇에게 구조화 task가 전달된다.
5. 전문 봇이 해당 agent web을 조작한다.
6. 결과물이 생성된다.
7. 결과물이 Cloud에 저장된다.
8. 결과물이 Telegram으로 전달된다.
9. phoenix command에서 상태와 결과를 볼 수 있다.
