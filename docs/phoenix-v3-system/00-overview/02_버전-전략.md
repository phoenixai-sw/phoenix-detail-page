# Phoenix V3 버전 전략

## 1. 전체 버전 관계

Phoenix 웹사이트 버전과 Phoenix Agent 버전은 서로 다르지만 V3에서 연결된다.

```text
웹사이트 V0   = 브라우저 저장 + 로컬 다운로드
웹사이트 V1   = Supabase Auth/DB/Storage
웹사이트 V1.5 = 결제/크레딧
웹사이트 V2   = 봇 자동화 준비
웹사이트 V3   = Agent Bot v2.0 연동

Agent v1.9 = 봇별 Telegram/OpenClaw 기본 운영
Agent v2.0 = agent web 컨트롤 스킬 추가
```

## 2. 웹사이트 단계

### V0

기본 작동 단계다.

특징:

1. API 키를 유저가 직접 입력
2. 브라우저 저장
3. 로컬 다운로드
4. 로그인 없음
5. Cloud 저장 없음

### V1

Cloud 저장 단계다.

추가:

1. Google 로그인
2. Supabase DB
3. Supabase Storage
4. 슈퍼 관리자 모드
5. 유저별 관리자 모드

### V1.5

결제/크레딧 단계다.

추가:

1. Toss 결제
2. 무료 크레딧
3. 유료 크레딧
4. 생성/편집 사용량 차감
5. 슈퍼 관리자 크레딧 관리

### V2

봇 자동화 준비 단계다.

추가:

1. 자동화용 `data-testid`
2. 안정적인 다운로드
3. 작업 상태 API
4. 봇 전용 경량 입력 흐름
5. 실패/재시도 상태값

### V3

Agent Bot v2.0 연동 단계다.

추가:

1. Genesis Bot 지휘
2. 전문 봇별 agent web 컨트롤
3. Telegram 결과 전달
4. phoenix command 상태판
5. Cloud 결과 통합 조회

## 3. Agent 단계

### Agent v1.9

현재 구조:

1. Genesis, Design, Writer, Video, Power Bot
2. Telegram 채널
3. OpenClaw profile 격리
4. PM2 실행
5. 봇별 역할 지정

### Agent v2.0

추가 목표:

1. Genesis Bot이 phoenix command 조작
2. Design Bot이 phoenix pages 조작
3. Design Bot이 phoenix slides 조작
4. Design Bot이 phoenix webs 조작
5. Design Bot이 phoenix images 조작
6. Writer Bot이 phoenix books 조작
7. Video Bot이 phoenix videos 조작
8. Power Bot이 phoenix reports 조작
9. Power Bot이 phoenix tax 조작
10. Power Bot이 phoenix dental 조작
11. Power Bot이 phoenix marketing 조작
12. 결과물을 Telegram으로 전달

## 4. 권장 진행 순서

```text
1. phoenix command 최소 버전
2. phoenix pages V1
3. phoenix pages V1.5
4. phoenix pages V2
5. Agent v2.0 Design Bot pages 스킬
6. phoenix images V0
7. Agent v2.0 Design Bot images 스킬
8. phoenix webs V0
9. Agent v2.0 Design Bot webs 스킬
10. phoenix books V0
11. Agent v2.0 Writer Bot books 스킬
12. phoenix slides V0
13. Agent v2.0 Design Bot slides 스킬
14. phoenix tax / dental / marketing V0
15. Agent v2.0 Power Bot 상담/대시보드 스킬
16. phoenix videos / reports 확장
17. 전체 V3 통합 테스트
```

## 5. 금지 사항

1. V1 없이 결제부터 붙이지 않는다.
2. V2 자동화 설계 없이 봇 조작부터 만들지 않는다.
3. Genesis Bot이 모든 agent web을 직접 조작하게 하지 않는다.
4. API 키와 Telegram token을 문서에 저장하지 않는다.
5. Agent v1.9 설치 코어를 불안정하게 크게 바꾸지 않는다.
