# phoenix command 코덱스용 작업지시서

## 1. 작업 목표

`phoenix command`를 Genesis Bot이 지휘하고 마스터가 전체 작업 상태를 볼 수 있는 최상위 웹사이트로 구축한다.

이 웹은 각 생성 웹의 기능을 중복 구현하지 않는다. 전문 agent web의 작업 상태와 결과를 모아 보여주는 지휘/관제 화면으로 만든다.

## 2. 관리 대상

```text
phoenix pages
phoenix slides
phoenix videos
phoenix reports
phoenix webs
phoenix books
phoenix images
phoenix tax
phoenix dental
phoenix marketing
```

## 3. 필수 화면

1. 전체 대시보드
2. 새 통합 작업 만들기
3. 작업 상세
4. 봇별 진행 상태
5. 결과물 모음
6. 상담/대시보드 상태
7. 관리자 설정

## 7. Genesis Bot 연동

Genesis Bot은 아래 형태로 구조화 작업을 등록한다.

```json
{
  "jobType": "multi_asset_campaign",
  "title": "상품 출시 패키지",
  "tasks": [
    { "target": "phoenix-pages", "bot": "design" },
    { "target": "phoenix-images", "bot": "design" },
    { "target": "phoenix-webs", "bot": "design" },
    { "target": "phoenix-slides", "bot": "design" },
    { "target": "phoenix-books", "bot": "writer" }
  ]
}
```

상담/대시보드 작업 예시:

```json
{
  "jobType": "consulting_dashboard",
  "title": "고객사 AI 검색 현황 분석",
  "tasks": [
    { "target": "phoenix-marketing", "bot": "power" }
  ]
}
```

## 8. 금지 사항

1. 각 전문 agent web 기능을 command 안에 중복 구현하지 않는다.
2. Genesis Bot이 모든 파일을 직접 생성하게 하지 않는다.
3. 실패 상태를 숨기지 않는다.
4. API 키와 Telegram token을 저장하지 않는다.
5. 상담형 결과를 전문가 검토 없이 확정 자문처럼 표시하지 않는다.

## 9. 1차 성공 기준

1. 통합 작업 생성
2. 작업 상태 표시
3. 봇별 task 표시
4. 결과물 링크 표시
5. 상담형 task 상태 표시
6. Genesis Bot 보고와 연결
