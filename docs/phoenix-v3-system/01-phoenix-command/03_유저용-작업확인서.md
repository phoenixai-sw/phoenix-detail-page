# phoenix command 유저용 작업확인서

## 1. 목적

`phoenix command`에서 전체 작업 상태와 결과물을 확인할 수 있는지 점검한다.

## 2. 접속 확인

1. phoenix command 주소로 접속한다.
2. 관리자 계정으로 로그인한다.
3. 대시보드가 보이는지 확인한다.

## 3. 상태판 확인

아래 항목이 보이는지 확인한다.

1. Genesis Bot 상태
2. Design Bot 상태
3. Writer Bot 상태
4. Video Bot 상태
5. Power Bot 상태
6. 최근 작업 목록
7. 실패 작업 목록
8. 상담/대시보드 작업 목록

## 4. 통합 작업 확인

예시 요청:

```text
신제품 출시용 상세페이지, 제품컷, 랜딩페이지, PPT, SNS 카피를 만들어줘.
```

확인할 것:

1. 작업 카드가 생성된다.
2. phoenix pages task가 생성된다.
3. phoenix images task가 생성된다.
4. phoenix webs task가 생성된다.
5. phoenix slides task가 생성된다.
6. phoenix books task가 생성된다.
7. 각 task의 상태가 업데이트된다.

## 5. 상담형 작업 확인

예시 요청:

```text
고객사의 AI 검색 노출 현황을 분석하고 상담 보고서를 만들어줘.
```

확인할 것:

1. phoenix marketing task가 생성된다.
2. Power Bot 상태가 표시된다.
3. 대시보드 업데이트 결과가 보인다.
4. 상담 요약과 보고서 링크가 보인다.

## 6. 결과물 확인

아래 결과가 링크로 표시되는지 확인한다.

1. 상세페이지 ZIP
2. 제품 이미지 PNG/JPEG
3. 랜딩페이지 HTML ZIP
4. PPTX/PDF
5. DOCX/PDF 또는 카피 문안
6. 상담 요약/보고서
7. Telegram 전달 기록

## 7. 실패 처리 확인

1. 일부 작업을 실패 상태로 만든다.
2. 실패 이유가 보이는지 확인한다.
3. 재시도 버튼 또는 마스터 확인 안내가 보이는지 확인한다.

## 8. 성공 기준

1. 여러 봇의 작업이 한 화면에서 보인다.
2. 결과물이 작업 단위로 묶인다.
3. 실패 상태가 숨겨지지 않는다.
4. Genesis Bot의 최종 보고와 화면 상태가 맞다.
