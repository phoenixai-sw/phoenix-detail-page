# phoenix slides 코덱스용 작업지시서

## 1. 작업 목표

`phoenix slides`를 PPT 생성 웹으로 신규 구축한다.

## 2. 권장 기술

현재 Next.js 기반을 유지한다.

추가 후보:

```text
pptxgenjs = PPTX 생성
pdf 변환 = 추후 서버 변환 또는 브라우저 이미지 변환
JSZip = 슬라이드 이미지 묶음 다운로드
```

## 3. 핵심 화면

1. 주제 입력 화면
2. 참고자료 업로드 화면
3. 슬라이드 조건 선택 화면
4. 생성 진행 화면
5. 슬라이드 미리보기 화면
6. 다운로드 화면

## 4. 자동화용 식별자

```text
data-testid="phoenix-slides-topic"
data-testid="phoenix-slides-upload"
data-testid="phoenix-slides-slide-count"
data-testid="phoenix-slides-tone"
data-testid="phoenix-slides-template"
data-testid="phoenix-slides-generate"
data-testid="phoenix-slides-preview"
data-testid="phoenix-slides-download-pptx"
data-testid="phoenix-slides-download-pdf"
```

## 5. API 후보

```text
src/app/api/slides/generate/route.ts
src/app/api/slides/export-pptx/route.ts
src/app/api/slides/export-pdf/route.ts
```

## 6. 데이터 구조

```text
slide_project
slide_sections
slide_assets
slide_exports
```

## 7. 금지 사항

1. 처음부터 복잡한 애니메이션 편집기를 만들지 않는다.
2. PPTX 생성과 디자인 편집을 한 번에 완성하려 하지 않는다.
3. 봇 자동화를 텍스트 라벨에만 의존하지 않는다.
4. 이미지 생성 비용이 큰 기능은 V1.5 크레딧과 연결한다.

## 8. 1차 성공 기준

1. 주제 입력 가능
2. 5~10장 슬라이드 구조 생성
3. PPTX 다운로드 가능
4. Design Bot이 자동으로 생성 흐름을 실행 가능
