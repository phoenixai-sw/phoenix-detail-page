# 봇간 지휘 프로토콜

## 1. 목적

Genesis Bot이 전문 봇에게 작업을 넘길 때 사용하는 공통 형식을 정의한다.

자연어만으로 일을 넘기면 누락과 오해가 생기므로, V3에서는 구조화된 task를 사용한다.

## 2. 기본 작업 구조

```json
{
  "jobId": "job_20260706_001",
  "taskId": "task_001",
  "requester": "genesis",
  "targetBot": "design",
  "targetWebsite": "phoenix-images",
  "action": "create_product_thumbnail",
  "priority": "normal",
  "requiresApproval": true,
  "inputs": {
    "brand": "Phoenix AI",
    "product": "sample product",
    "purpose": "thumbnail",
    "format": "png"
  },
  "delivery": {
    "telegram": true,
    "cloudSave": true,
    "zip": false
  }
}
```

phoenix-command

## 6. 전문 봇 응답 구조

```json
{
  "jobId": "job_20260706_001",
  "taskId": "task_001",
  "targetBot": "design",
  "status": "completed",
  "message": "제품 썸네일 PNG를 생성하고 전달했습니다.",
  "results": [
    {
      "type": "image",
      "name": "thumbnail.png",
      "url": "cloud-or-local-path"
    }
  ],
  "error": null
}
```

## 7. 실패 응답 구조

```json
{
  "jobId": "job_20260706_001",
  "taskId": "task_001",
  "targetBot": "power",
  "status": "failed",
  "message": "상담 요약 문서 생성에 실패했습니다.",
  "results": [],
  "error": {
    "code": "document_export_failed",
    "retryable": true,
    "suggestedAction": "상담 요약을 text로 먼저 저장한 뒤 PDF를 다시 생성"
  }
}
```

## 8. 승인 필요 상황

1. 유료 크레딧 사용량이 큰 작업
2. 영상 생성
3. 외부 배포
4. 여러 agent web 동시 실행
5. 프로젝트 삭제
6. 세무/의료 상담 결과 외부 전달

## 9. 금지 사항

1. taskId 없이 작업하지 않는다.
2. 실패한 작업을 completed로 표시하지 않는다.
3. API 키나 token을 payload에 넣지 않는다.
4. Telegram chat id를 공개 문서에 저장하지 않는다.
5. 전문 봇의 역할과 작업을 강제로 섞지 않는다.
