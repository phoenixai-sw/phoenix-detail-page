# V2(유저수파베이스연결) 코덱스용 작업지시서

## 1. 작업 목표

V2는 고급 유저가 Phoenix가 운영하는 Supabase가 아니라, 자기 Supabase 프로젝트에 상세페이지 결과를 저장할 수 있게 만드는 단계다.

대상 유저:

```text
강사
대행사
기업
개발 이해도가 있는 고급 유저
자기 데이터와 저장 비용을 직접 관리하려는 유저
```

## 2. V2 핵심 구조

```text
Phoenix detail page = 생성/편집 도구
유저 Supabase Storage = 이미지 파일 저장
유저 Supabase Database = 프로젝트 정보 저장
```

V2는 일반 입문자 기본 기능이 아니다.
설정 화면에서 고급 옵션으로 분리한다.

## 3. 보안 원칙

가장 중요한 원칙:

```text
service role key를 브라우저에 입력하게 하지 않는다.
```

허용 가능한 값:

1. Supabase Project URL
2. Supabase anon key
3. Storage bucket 이름

주의가 필요한 값:

```text
service role key
database password
JWT secret
```

이 값들은 브라우저에 저장하거나 입력받지 않는다.

## 4. 유저 Supabase 사전 준비

고급 유저가 직접 준비해야 할 것:

1. Supabase 프로젝트
2. Storage 버킷
3. Database 테이블
4. RLS 정책
5. anon key
6. Project URL

권장 버킷:

```text
detail-page-images
```

권장 테이블:

```text
external_projects
external_project_sections
external_project_images
```

## 5. 앱 UI 구현 범위

추가할 메뉴:

```text
고급 저장 설정
```

설정 화면 입력 항목:

1. 저장 방식
   - 브라우저 저장
   - Phoenix cloud 저장
   - user supabase 연결

2. Supabase Project URL

3. Supabase anon key

4. Storage bucket 이름

5. 연결 테스트 버튼

6. 설정 저장 버튼

7. 설정 초기화 버튼

## 6. 연결 테스트

추가 API 또는 클라이언트 기능:

```text
Supabase URL 유효성 확인
anon key 유효성 확인
Storage bucket 접근 확인
필요 테이블 접근 확인
RLS 정책으로 insert 가능한지 확인
```

연결 테스트 성공 메시지:

```text
user supabase 연결이 확인되었습니다.
```

실패 시 메시지:

```text
Supabase URL 또는 anon key를 확인해주세요.
Storage bucket 접근 권한을 확인해주세요.
필요 테이블 또는 RLS 정책을 확인해주세요.
```

## 7. 저장 흐름

유저가 `user supabase 연결`을 선택한 경우:

```text
1. 생성/편집 결과 이미지 확보
2. 이미지 dataUrl을 Blob으로 변환
3. 유저 Supabase Storage에 업로드
4. 업로드된 path 또는 public URL 확보
5. 유저 Supabase Database에 프로젝트 정보 저장
6. 저장 성공 메시지 표시
```

실패 시:

```text
브라우저 저장은 유지한다.
user supabase 저장 실패 메시지를 보여준다.
생성 결과를 잃지 않게 한다.
```

## 8. V2에서 유지할 것

1. V0 브라우저 저장
2. V0 로컬 다운로드
3. V1 Phoenix cloud 저장
4. 유저 API 키 입력 방식

V2는 기존 기능을 대체하지 않고 고급 옵션으로 추가한다.

## 9. V2에서 금지할 것

1. service role key 브라우저 저장
2. 유저 Supabase 비밀번호 저장
3. Phoenix Supabase와 유저 Supabase 데이터 혼합
4. 일반 유저 화면을 복잡하게 만들기
5. 연결 실패 시 생성 결과 삭제

## 10. 필요한 파일 예시

```text
src/lib/external-supabase.ts
src/app/api/external-supabase/health/route.ts
src/components/redesign-wizard.tsx
```

가능하면 Supabase 클라이언트는 고급 설정이 켜졌을 때만 초기화한다.

## 11. 성공 기준

1. 고급 저장 설정 메뉴가 보인다.
2. user supabase 연결 옵션을 선택할 수 있다.
3. Supabase URL과 anon key를 입력할 수 있다.
4. 연결 테스트가 가능하다.
5. 유저 Storage에 이미지가 저장된다.
6. 유저 Database에 프로젝트 정보가 저장된다.
7. Phoenix Supabase에는 저장되지 않는다.
8. 설정 초기화 시 브라우저 저장 방식으로 돌아간다.

## 12. 마스터에게 확인할 것

구현 전에 아래를 확인한다.

1. V2를 일반 메뉴에 노출할지
2. 고급 설정 안에 숨길지
3. 유저 Supabase 템플릿 SQL을 제공할지
4. Storage public bucket을 사용할지 signed URL을 사용할지
5. anon key만으로 가능한 정책을 만들지
