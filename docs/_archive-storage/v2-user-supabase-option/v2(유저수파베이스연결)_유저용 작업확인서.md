# V2(유저수파베이스연결) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 고급 유저가 자기 Supabase 프로젝트를 Phoenix detail page에 연결해 이미지와 프로젝트를 저장할 수 있는지 확인하는 문서다.

V2는 입문자 기본 기능이 아니다.
Supabase 프로젝트를 직접 만들고 관리할 수 있는 유저를 위한 고급 옵션이다.

## 2. 준비물

1. 본인 Supabase 계정
2. 본인 Supabase 프로젝트
3. Supabase Project URL
4. Supabase anon key
5. Storage bucket
6. 프로젝트 저장용 Database 테이블
7. RLS 정책
8. OpenAI 또는 Google API 키
9. 테스트용 상세페이지 이미지 또는 PDF

## 3. Supabase 프로젝트 준비

### 3.1 Supabase 접속

아래 사이트에 접속한다.

```text
https://supabase.com/dashboard
```

### 3.2 프로젝트 생성

1. `New project`를 누른다.
2. 프로젝트 이름을 입력한다.
3. Database password를 설정한다.
4. Region을 선택한다.
5. 프로젝트 생성을 완료한다.

### 3.3 Project URL 확인

Supabase 프로젝트에서 아래 메뉴로 이동한다.

```text
Project Settings
API
```

확인할 값:

```text
Project URL
```

### 3.4 anon key 확인

같은 화면에서 아래 값을 확인한다.

```text
anon public key
```

주의:

```text
service_role key는 Phoenix detail page 화면에 입력하지 않는다.
```

## 4. Storage bucket 준비

Supabase에서 아래 메뉴로 이동한다.

```text
Storage
```

1. `New bucket`을 누른다.
2. 버킷 이름을 입력한다.

권장 이름:

```text
detail-page-images
```

3. public/private 정책을 선택한다.
4. V2 가이드에서 요구하는 RLS 정책을 적용한다.

확인할 것:

```text
버킷이 생성되어야 한다.
이미지 업로드 권한이 있어야 한다.
```

## 5. Database 테이블 준비

Supabase에서 아래 메뉴로 이동한다.

```text
Table Editor
```

또는 SQL Editor를 사용한다.

필요 테이블 예시:

1. `external_projects`
2. `external_project_sections`
3. `external_project_images`

각 테이블은 최소한 아래 정보를 저장할 수 있어야 한다.

```text
project id
project title
section id
image path
created at
updated at
```

## 6. Phoenix detail page에서 고급 저장 설정 열기

Phoenix detail page 사이트에 접속한다.

작업 화면에서 아래 메뉴를 찾는다.

```text
고급 저장 설정
```

메뉴가 보이지 않으면:

1. 설정 메뉴를 연다.
2. 고급 옵션 표시가 있는지 확인한다.
3. 권한이 필요한 기능인지 확인한다.

## 7. 저장 방식 선택

고급 저장 설정에서 저장 방식을 선택한다.

선택할 값:

```text
user supabase 연결
```

다른 옵션:

```text
브라우저 저장
Phoenix cloud 저장
```

## 8. Supabase 정보 입력

아래 값을 입력한다.

1. Supabase Project URL
2. Supabase anon key
3. Storage bucket 이름

입력 예시:

```text
Project URL: https://xxxx.supabase.co
anon key: eyJ...
bucket: detail-page-images
```

입력하지 말아야 할 값:

```text
service_role key
database password
JWT secret
```

## 9. 연결 테스트

`연결 테스트` 버튼을 누른다.

성공 기준:

```text
user supabase 연결이 확인되었습니다.
```

실패 시 확인:

1. Project URL이 맞는가
2. anon key가 맞는가
3. bucket 이름이 맞는가
4. RLS 정책이 insert를 허용하는가
5. 필요한 테이블이 생성되어 있는가

## 10. 설정 저장

연결 테스트가 성공하면 `설정 저장`을 누른다.

성공 기준:

```text
고급 저장 설정을 저장했습니다.
```

## 11. 상세페이지 생성

1. `새 작업`으로 이동한다.
2. 이미지 또는 PDF를 업로드한다.
3. 제작 엔진을 선택한다.
4. 요청 문구를 입력한다.
5. 1장을 생성한다.

성공 기준:

```text
결과 이미지가 생성된다.
```

## 12. user supabase 저장

결과 화면에서 저장 버튼을 누른다.

예상 버튼:

```text
저장
클라우드 저장
user supabase 저장
```

성공 기준:

```text
user supabase에 저장했습니다.
```

또는 비슷한 저장 완료 메시지가 표시된다.

## 13. Supabase Storage 확인

본인 Supabase Dashboard로 돌아간다.

이동:

```text
Storage
detail-page-images
```

확인할 것:

1. 프로젝트 폴더가 생겼는가
2. 섹션 이미지가 업로드됐는가
3. 수정 이미지가 업로드됐는가

## 14. Supabase Database 확인

이동:

```text
Table Editor
```

확인할 테이블:

1. `external_projects`
2. `external_project_sections`
3. `external_project_images`

확인할 것:

1. 프로젝트 제목이 저장됐는가
2. 이미지 path가 저장됐는가
3. 생성 시간이 저장됐는가

## 15. 설정 초기화 확인

고급 저장 설정에서 `설정 초기화`를 누른다.

성공 기준:

```text
저장 방식이 브라우저 저장으로 돌아간다.
```

초기화 후에는 user supabase 저장을 시도하지 않아야 한다.

## 16. 실패 시 복구

저장이 실패해도 생성 결과는 화면에 남아 있어야 한다.

확인할 것:

1. 이미지 다운로드가 가능한가
2. 브라우저 저장이 가능한가
3. 다시 Supabase 설정을 수정할 수 있는가

## 17. V2 성공 기준

1. 고급 저장 설정 메뉴가 보인다.
2. user supabase 연결을 선택할 수 있다.
3. Project URL을 입력할 수 있다.
4. anon key를 입력할 수 있다.
5. Storage bucket을 입력할 수 있다.
6. 연결 테스트가 성공한다.
7. 생성 이미지가 본인 Supabase Storage에 저장된다.
8. 프로젝트 정보가 본인 Supabase Database에 저장된다.
9. service role key를 입력하지 않아도 된다.
10. 설정 초기화 후 브라우저 저장으로 돌아간다.
