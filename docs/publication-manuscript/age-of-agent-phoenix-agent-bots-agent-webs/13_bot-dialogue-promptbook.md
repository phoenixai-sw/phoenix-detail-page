# 봇별 대화 시나리오와 프롬프트북

## 1. 이 문서의 목적

이 문서는 출판 원고 안에 넣을 수 있는 실제 대화 예시와 프롬프트를 정리한다.

집필 모델은 이 문서를 활용해 독자가 “봇에게 어떻게 말해야 하는지”를 구체적으로 이해하게 만든다.

## Genesis Bot

### 역할 요약

Genesis Bot은 Phoenix 시스템에서 아래 작업을 담당한다.

1. 마스터 요청 해석
2. 작업 분해
3. phoenix command 등록
4. 전문 봇 라우팅
5. 상태 추적
6. 최종 보고

### 기본 요청 예시

```text
이 제품으로 상세페이지, 제품컷, 랜딩페이지, PPT, SNS 카피까지 만드는 작업을 나눠줘.
필요한 agent web과 담당 봇을 구분하고, 실행 전에 승인 필요한 항목을 알려줘.
```

### 좋은 답변에 포함되어야 할 요소

1. 요청을 이해했는지 짧게 확인한다.
2. 필요한 입력값이 부족하면 질문한다.
3. 담당 agent web을 명시한다.
4. 승인 필요한 작업을 구분한다.
5. 결과 저장 위치를 outputs 기준으로 안내한다.
6. Telegram 전송은 선택 사항으로 묻는다.

### 나쁜 답변 유형

1. 승인 없이 유료 작업을 시작한다.
2. token이나 API key를 화면에 출력하라고 요구한다.
3. 결과물이 없는데 완료라고 말한다.
4. 세무/의료 판단을 확정적으로 말한다.
5. 어떤 agent web을 쓸지 밝히지 않는다.

### 대화 시나리오

#### 시나리오 1

마스터:

```text
Genesis Bot, 지금 작업 준비 상태를 확인하고 필요한 입력값을 질문해줘.
```

Genesis Bot:

```text
확인했습니다. 먼저 작업 목적과 결과물 형식을 확인하겠습니다.
담당 agent web을 확인하고, 승인 필요한 항목이 있으면 실행 전에 멈추겠습니다.
결과물은 먼저 outputs에 저장하고, Telegram 전송은 원하실 때 진행하겠습니다.
```

집필 포인트:

이 대화는 봇이 무작정 실행하지 않고, 준비 상태와 승인 조건을 먼저 확인하는 태도를 보여준다.

```text
Power Bot, 지금 작업 준비 상태를 확인하고 필요한 입력값을 질문해줘.
```

Power Bot:

```text
확인했습니다. 먼저 작업 목적과 결과물 형식을 확인하겠습니다.
담당 agent web을 확인하고, 승인 필요한 항목이 있으면 실행 전에 멈추겠습니다.
결과물은 먼저 outputs에 저장하고, Telegram 전송은 원하실 때 진행하겠습니다.
```

집필 포인트:

이 대화는 봇이 무작정 실행하지 않고, 준비 상태와 승인 조건을 먼저 확인하는 태도를 보여준다.
