# AI 집필 모델용 프롬프트 지침

## 1. 이 문서의 목적

이 문서는 GPT 5.5, Claude Opus 4.8, Fable에게 원고 집필을 맡길 때 사용할 프롬프트와 품질 기준을 정리한다.

각 모델은 문체가 다르므로, 같은 기준을 주어도 결과가 달라질 수 있다. 이 문서는 모델 간 편차를 줄이고, 책 전체의 일관성을 유지하기 위한 기준이다.

## 장 확장 프롬프트

```text
아래 장의 핵심을 입문자용 출판 원고로 확장해라. 개념 설명, 실제 예시, 운영상 주의점, 실습 체크리스트를 포함하라. 의미 없는 반복은 금지한다.
```

### 사용 시 주의

1. 원본의 핵심 사실을 바꾸지 않는다.
2. 날짜, 가격, 모델명처럼 변할 수 있는 정보는 단정하지 않는다.
3. 공식 문서 확인이 필요한 부분은 확인 필요로 남긴다.
4. token, API key, OAuth credential 예시는 실제값처럼 쓰지 않는다.

## 7. 모델별 기대 역할

### GPT 5.5

구조화와 절차형 설명에 강점을 두고 활용한다. 장별 목차, 실습 순서, 체크리스트, 오류 대응 표를 만드는 역할을 맡긴다.

### Claude Opus 4.8

긴 문단의 자연스러운 서사와 개념 설명을 확장하는 역할을 맡긴다. 다만 과도한 비유가 늘어나지 않도록 실전 절차와 체크리스트를 함께 요구한다.

### Fable

문장 리듬, 사례 서술, 독자 친화적 설명을 보강하는 역할을 맡긴다. 기술 사실은 검증 자료와 대조하도록 지시한다.

## 8. 최종 합본 원칙

1. 한 모델의 원고를 그대로 최종본으로 쓰지 않는다.
2. GPT 5.5의 구조, Claude의 설명력, Fable의 문장감을 조합한다.
3. 최종본은 안전 검수 체크리스트를 통과해야 한다.
4. 공식 문서 근거가 필요한 부분은 리서치 노트와 대조한다.
5. 모든 Word 파일은 Markdown 원본과 함께 관리한다.
