# 원고 개요

## 제목

```text
age of agent - phoenix agent bots & agent webs
```

## 원고의 핵심 질문

앞으로의 웹사이트는 사람이 버튼을 누르는 도구로만 남을까?

아니면 에이전트 봇이 사람 대신 접속하고, 판단하고, 입력하고, 생성하고, 다운로드하고, 상담하고, 보고하는 업무 공간이 될까?

이 원고는 두 번째 가능성을 다룬다.

## 핵심 주장

에이전트 시대의 웹사이트는 단순한 화면이 아니다.

좋은 웹사이트는 사람이 직접 사용할 수 있어야 한다. 동시에 에이전트 봇도 안정적으로 사용할 수 있어야 한다.

이런 웹사이트를 이 원고에서는 `agent webs`라고 부른다.

Phoenix 구조에서는 `phoenix command`가 최상위 지휘 웹이 되고, 각 전문 agent web은 그 아래에서 봇에 의해 컨트롤된다.

## Phoenix 시스템의 구성

Phoenix 시스템은 크게 두 축으로 구성된다.

1. Phoenix Agent Bots
   - Genesis Bot
   - Design Bot
   - Writer Bot
   - Video Bot
   - Power Bot

2. Phoenix Agent Webs
   - phoenix command: 전체 지휘/상태 관리 웹
   - phoenix pages: 상세페이지 생성 웹
   - phoenix slides: PPT 생성 웹
   - phoenix videos: 영상 생성 웹
   - phoenix reports: 리서치/보고서 생성 웹
   - phoenix webs: 웹사이트/랜딩페이지 HTML 생성 웹
   - phoenix books: 강의교안/SNS 카피라이팅 생성 웹
   - phoenix images: 제품 목업/제품컷/카드뉴스/썸네일 생성 웹
   - phoenix tax: 양도소득세/세무사 정보 및 채팅 상담 웹
   - phoenix dental: 치아관리/치과병원 정보 및 채팅 상담 웹
   - phoenix marketing: AI 검색 순위/인용 정보/예상 질문 대시보드 및 채팅 상담 웹

## 현재 상태

현재 실제로 구축되어 있는 것은 `phoenix pages`다.

`phoenix pages`는 상세페이지 생성 웹이다. 현재는 V0 상태로 둔다. 즉 로그인, DB, Storage, 결제, 봇 자동화까지 완성된 상태는 아니다.

다만 출판원고에서는 앞으로의 진화 방향을 설명한다.

이번 단계에서는 실제 웹사이트 코드나 봇 코드를 수정하지 않는다.

작업 대상은 출판 원고뿐이다. 목표는 원고의 양과 품질을 높이고, 입문자가 따라 할 수 있는 실습과 운영 체크리스트를 추가하는 것이다.

## 실전 안내 기준

이 원고는 입문자용으로 작성한다.

독자는 아래 흐름을 따라 할 수 있어야 한다.

1. 회사 GitHub 저장소에서 웹사이트 파일을 받는다.
2. `phoenix pages`는 Vercel에 배포한다.
3. 나머지 agent web은 Netlify에 배포한다.
4. Phoenix Agent v2.0은 `https://github.com/phoenixai-sw/phoenix-agents-camp.git`에서 받는다.
5. Telegram에서는 `genesis_bot`, `power_bot`, `design_bot`, `video_bot`, `writer_bot`을 기준으로 연결한다.

## 독자에게 전달할 메시지

1. 에이전트 봇은 단순한 채팅창이 아니라 업무 실행자다.
2. 웹사이트는 사람이 보는 화면이면서 봇이 조작하는 작업장이 될 수 있다.
3. phoenix command는 그 작업장을 지휘하는 최상위 웹이다.
4. agent webs는 봇과 웹의 관계를 연결한다.
5. Phoenix는 봇과 웹을 함께 설계하는 시스템이다.
