# 용어 정리

## age of agent

AI가 단순히 답변하는 것을 넘어, 사람의 일을 대신 실행하는 시대로 이동하고 있다는 의미다.

## Phoenix Agent Bots

Phoenix 시스템 안에서 역할을 가진 에이전트 봇들이다.

예시:

1. Genesis Bot
2. Design Bot
3. Writer Bot
4. Video Bot
5. Power Bot

## Agent Webs

사람이 직접 사용할 수도 있고, 에이전트 봇이 대신 컨트롤할 수도 있는 웹사이트다.

## phoenix command

전체 작업을 지휘하고 상태를 확인하는 최상위 웹사이트다.

Genesis Bot의 중심 화면이 된다.

## phoenix pages

상세페이지 생성 웹이다.

현재 실제로 구축된 `phoenix detail page`를 앞으로 `phoenix pages`로 발전시킨다.

## phoenix slides

PPT 생성 웹이다.

Design Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix videos

영상 생성 웹이다.

Video Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix reports

리서치/보고서 생성 웹이다.

Power Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix webs

웹사이트/랜딩페이지 HTML 생성 웹이다.

Design Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix books

강의교안, SNS 카피라이팅, 카드뉴스 문안, 출판 원고 초안을 생성하는 웹이다.

Writer Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix images

제품 목업, 제품컷, 카드뉴스, 썸네일을 생성하는 이미지 제작 웹이다.

Design Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix tax

양도소득세 관련 정보, 세무사 사무실 정보, 세무사님 소개, agent 채팅상담을 제공하는 웹이다.

Power Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix dental

치아관리 정보, 치과병원 정보, 치과원장님 소개, agent 채팅상담을 제공하는 웹이다.

Power Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## phoenix marketing

고객사별 AI 검색 순위, AI 플랫폼별 인용 정보, 예상 AI 검색 질문을 관리하는 대시보드와 채팅 상담 웹이다.

Power Bot이 컨트롤할 수 있는 웹사이트로 구상한다.

## Genesis Bot

비서실장 역할의 봇이다.

마스터의 요청을 이해하고, 일을 나누고, 전문 봇에게 지시하고, 최종 결과를 보고한다.

## Design Bot

시각 작업 담당 봇이다.

상세페이지, PPT, 랜딩페이지, 제품 이미지, 카드뉴스, 썸네일을 다룬다.

## Writer Bot

문서와 카피라이팅 담당 봇이다.

강의교안, SNS 카피, 카드뉴스 문안, 출판 원고 초안, 실습 과제를 다룬다.

## Video Bot

영상 작업 담당 봇이다.

숏폼, 광고 영상, 장면 구성, 영상 프롬프트, MP4 결과물을 다룬다.

## Power Bot

리서치, 상담, 대시보드 담당 봇이다.

시장조사, 보고서, 세무 상담, 치과 상담, AI 검색 마케팅 분석을 다룬다.

## V0

현재 기본 작동 상태다.

로그인, DB, Storage, 결제, 봇 자동화 없이 사람이 직접 사용할 수 있는 웹 상태를 의미한다.

## V1

Cloud 저장 단계다.

Supabase Auth, Database, Storage가 붙는다.

## V1.5

결제와 크레딧 단계다.

Toss 결제, 무료 크레딧, 유료 크레딧, 사용량 차감이 붙는다.

## V2

봇 자동화 준비 단계다.

웹사이트에 봇이 조작하기 쉬운 구조와 상태 확인 기능을 추가한다.

## V3

Agent Bot v2.0과 웹사이트가 연동되는 단계다.

봇이 웹사이트를 컨트롤하고, 봇끼리 작업을 주고받는다.

## Phoenix Agent v2.0

웹컨트롤형 실행 에이전트 버전이다.

기존 v1.9의 Telegram/OpenClaw 기반 운영 구조를 보존하면서, 각 봇이 담당 agent web을 조작할 수 있도록 web control skill을 추가한다.

## Phoenix Agent v2.2

Master Builder Agent 단계다.

봇이 작업 경험을 바탕으로 스킬 개선안을 만들고, Genesis Bot이 이를 정리하며, 마스터 승인 후 updater로 반영하는 승인형 재귀개선 단계다.

## Playwright

브라우저를 자동으로 조작하는 도구다.

사람이 버튼을 누르고 입력창에 글을 쓰는 것처럼, 코드와 에이전트가 웹 화면을 조작할 수 있게 해준다.

## Playwright MCP

에이전트가 Playwright를 사용할 수 있도록 연결해주는 방식이다.

Phoenix Agent v2.0에서는 agent web 조작 가능성을 검증하고, 안정적인 반복 실행 흐름을 만들기 위한 기반으로 사용한다.

## data-testid

봇이 웹 화면의 버튼, 입력창, 상태 표시, 다운로드 링크를 안정적으로 찾기 위한 이름표다.

화면 디자인이 조금 바뀌어도 `data-testid`가 유지되면 봇은 같은 요소를 찾을 수 있다.

## structured task

Genesis Bot이 전문 봇에게 일을 넘길 때 사용하는 구조화된 작업 지시다.

작업 번호, 담당 봇, 대상 웹사이트, 실행 action, 입력값, 승인 필요 여부, 결과 전달 방식을 함께 담는다.

## jobId

하나의 큰 작업을 구분하는 번호다.

예를 들어 “한 제품으로 상세페이지, 제품컷, 랜딩페이지, PPT까지 만들기” 전체가 하나의 job이 될 수 있다.

## taskId

job 안에서 나뉜 개별 작업 번호다.

상세페이지 생성, 제품컷 생성, 랜딩페이지 생성, PPT 생성은 각각 별도의 task가 될 수 있다.

## outputs

Phoenix Agent가 결과물을 기본으로 저장하는 로컬 폴더다.

결과물은 먼저 outputs에 저장하고, Telegram 전송은 유저에게 물어본 뒤 진행한다.

## Telegram 선택 전송

결과물을 자동으로 Telegram에 보내지 않고, 유저가 원할 때만 보내는 방식이다.

파일 용량, 검토 필요성, 외부 전달 위험을 고려하기 위한 운영 원칙이다.

## 마스터 승인

비용, 외부 공개, 삭제, 세무/의료 상담 결과 전달처럼 위험하거나 되돌리기 어려운 작업 전에 마스터가 확인하는 절차다.

## installer

Phoenix Agent를 새로 설치하는 파일 묶음이다.

Windows 11은 주로 `.ps1`, macOS는 `.command`와 `.sh` 파일을 사용한다.

## updater

기존 설치본을 새 버전으로 올리는 파일 묶음이다.

v2.0 updater는 v1.9의 token, chat id, outputs, logs, PCS/PTS 스킬을 보존하면서 web control skill을 추가해야 한다.

## cleaner

로컬 봇과 관련 파일을 정리하거나 삭제하는 파일 묶음이다.

outputs 삭제 전에는 백업 여부를 물어야 하며, GitHub 저장소나 외부 배포물은 cleaner가 삭제하지 않는다.
