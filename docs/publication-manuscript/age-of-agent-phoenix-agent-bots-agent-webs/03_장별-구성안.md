# 장별 구성안

## 1장. 에이전트 시대가 온다

핵심 내용:

1. AI는 답변 도구에서 실행 도구로 이동한다.
2. 에이전트는 업무를 받아 실제 행동을 한다.
3. 웹사이트의 역할도 바뀐다.

## 2장. agent webs란 무엇인가

핵심 내용:

1. 사람이 쓰는 웹과 봇이 조작하는 웹의 결합
2. 기존 웹사이트와의 차이
3. 자동화를 위한 화면 설계 기준

## 3장. phoenix command

핵심 내용:

1. phoenix command는 최상위 지휘 웹이다.
2. Genesis Bot과 마스터가 전체 작업을 본다.
3. agent web은 command 아래에서 실행 계층으로 움직인다.

## 4장. Phoenix Agent Bots

핵심 내용:

1. Genesis Bot은 전체 지휘를 맡는다.
2. Design Bot은 시각 결과물을 만든다.
3. Writer Bot은 문서와 카피를 만든다.
4. Video Bot은 영상을 만든다.
5. Power Bot은 리서치, 상담, 대시보드를 맡는다.

## 6장. V0에서 V3로 가는 길

핵심 내용:

1. V0: 사람이 직접 쓰는 기본 웹
2. V1: 로그인, DB, Storage
3. V1.5: 결제와 크레딧
4. V2: 봇 자동화 준비
5. V3: 봇과 웹의 실전 연결

## 7장. 실제 작업 흐름

핵심 내용:

1. 마스터가 Genesis Bot에게 요청한다.
2. Genesis Bot이 phoenix command에 작업을 등록한다.
3. Genesis Bot이 작업을 나눈다.
4. 전문 봇들이 각 agent web을 컨트롤한다.
5. phoenix command가 상태판 역할을 한다.
6. 결과물이 Telegram과 Cloud로 전달된다.

## 8장. 상담형 agent web

핵심 내용:

1. phoenix tax
2. phoenix dental
3. phoenix marketing
4. 고위험 정보 안전 표현
5. 전문가 확인 필요성

## 9장. 사람과 봇이 함께 일하는 방식

핵심 내용:

1. 사람은 방향과 승인에 집중한다.
2. 봇은 반복 실행과 결과 정리에 집중한다.
3. 웹사이트는 그 둘을 연결하는 작업장이 된다.

## 10장. GitHub에서 웹사이트를 받아 배포하기

7. 운영 중 장애를 확인하기
