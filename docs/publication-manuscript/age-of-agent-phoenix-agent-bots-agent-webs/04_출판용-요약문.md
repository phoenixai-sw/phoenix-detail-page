# 출판용 요약문

AI는 더 이상 대답만 하는 도구에 머물지 않는다. 에이전트 시대의 AI는 사람 대신 접속하고, 판단하고, 입력하고, 생성하고, 상담하고, 다운로드하고, 보고하는 실행자가 된다.

이 변화는 웹사이트의 의미를 바꾼다. 앞으로의 웹사이트는 사람이 직접 쓰기 쉬워야 할 뿐 아니라, 에이전트 봇이 안정적으로 조작할 수 있어야 한다. 이 책에서는 그런 웹사이트를 `agent webs`라고 부른다.

Phoenix 시스템에서는 `phoenix command`가 최상위 지휘 웹이 된다. 그 아래에 상세페이지 생성 웹 `phoenix pages`, PPT 생성 웹 `phoenix slides`, 영상 생성 웹 `phoenix videos`, 보고서 생성 웹 `phoenix reports`, 랜딩페이지 HTML 생성 웹 `phoenix webs`, 강의교안/SNS 카피 생성 웹 `phoenix books`, 이미지 생성 웹 `phoenix images`, 세무 상담 웹 `phoenix tax`, 치과 상담 웹 `phoenix dental`, AI 검색 마케팅 대시보드 `phoenix marketing`이 연결된다.

이 웹사이트들은 개별로도 작동하지만, Phoenix Agent Bot에 의해 컨트롤될 수도 있다. Genesis Bot은 전체 작업을 지휘하고, Design Bot은 상세페이지, PPT, 랜딩페이지, 이미지를 만들고, Writer Bot은 강의교안과 카피를 만들고, Video Bot은 영상을 만들고, Power Bot은 보고서, 상담, 대시보드를 다룬다.

`age of agent - phoenix agent bots & agent webs`는 사람과 봇이 함께 일하는 새로운 업무 구조를 설명하는 원고다.

이 책은 개념 설명에서 멈추지 않는다. 독자는 GitHub에서 웹사이트 파일을 받고, 로컬에서 실행하고, Vercel 또는 Netlify에 배포하고, Telegram BotFather로 봇을 만들고, Phoenix Agent v2.0 설치 파일을 준비하는 흐름까지 따라 해볼 수 있다.

특히 Phoenix Agent v2.0은 `Web Control Agent` 단계로 설명한다. 봇이 담당 agent web을 안정적으로 조작하기 위해 Playwright MCP, `data-testid`, 구조화 task, outputs 저장, Telegram 선택 전송, 마스터 승인 정책이 왜 필요한지 입문자 눈높이에서 다룬다.

핵심 메시지는 단순하다.

```text
Agent Webs는 사람이 쓰는 화면이자, 봇이 일하는 작업장이다.
```
