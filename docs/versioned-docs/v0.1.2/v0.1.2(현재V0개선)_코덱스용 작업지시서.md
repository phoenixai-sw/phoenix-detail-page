# v0.1.2 코덱스용 작업지시서

## 1. 목적

이 문서는 V1으로 넘어가기 전, 현재 V0 서비스에 누적된 기능 개선을 유지보수하기 위한 코덱스 작업 지시서이다.

V0.1.2의 목표는 브라우저 저장 기반 상세페이지 생성기를 안정적으로 운영하면서, V1 클라우드 저장으로 넘어갈 준비를 하는 것이다.

## 2. 현재 기능 범위

코덱스는 다음 기능을 V0.1.2 기준 기능으로 취급해야 한다.

1. 원본 이미지/PDF 업로드
2. OpenAI Image 2.0 생성
3. Google Nano Banana 2 생성
4. 1장, 4장, 8장 생성 옵션
5. 9:16, 1:1, 4:5 출력 비율
6. 선택 비율에 맞춘 결과 이미지 정리
7. 결과 저장
8. 최근 제작 작업 목록
9. 최근 작업 제목 수정
10. ZIP 다운로드
11. 개별 이미지 다운로드
12. 개별 이미지 편집
13. AI 멘트 생성
14. 페이지 추가 생성
15. 최대 12장 제한
16. S11/S12 추가 템플릿
17. 모델/인물 이미지 홀수 페이지 중심 노출 규칙
18. 원본 자료가 없을 때 기존 결과 이미지 참고 생성
19. 기존 결과 이미지 참고 생성 시 압축 전송

## 3. 주요 파일

- `src/components/redesign-wizard.tsx`
  - 대부분의 UI, 생성 흐름, 저장, 다운로드, 추가 생성, 개별 편집 호출 담당
- `src/app/api/generate/route.ts`
  - 이미지 생성 API
  - 섹션 템플릿
  - 최대 12장 제한
  - 출력 비율 프롬프트
- `src/app/api/edit-section/route.ts`
  - 개별 이미지 편집 API
- `src/app/api/ai-comment/route.ts`
  - AI 멘트 생성 API
- `docs/versioned-docs/v0.1.2`
  - 현재 V0.1.2 기준 문서

## 4. 작업 원칙

1. V1 이전 기능 개선은 V0.x 문서에 누적 반영한다.
2. 저장 방식이 바뀌는 작업은 V1 문서와 충돌 여부를 확인한다.
3. 브라우저 저장 기반 기능은 V1 전환 전까지 깨지지 않게 유지한다.
4. 추가 생성은 결과함에서 유지되어야 하며 제작실로 강제 이동하면 안 된다.
5. 원본 업로드 파일이 없더라도 기존 결과 이미지가 있으면 참고 이미지로 추가 생성을 시도해야 한다.
6. 기존 결과 이미지를 참고 이미지로 보낼 때는 반드시 압축해 요청 본문 크기를 줄인다.
7. 커밋/푸시는 사용자가 명시적으로 요청하지 않는 한 실행하지 않는다.

## 5. V0.1.2 검증 체크리스트

코드 수정 후 다음을 확인한다.

1. `npm run build` 성공
2. `/studio` 로컬 응답 확인
3. 생성 페이지 옵션 표시 확인
4. 출력 비율 옵션 표시 확인
5. ZIP 다운로드 함수 영향 확인
6. 페이지 추가 생성 카드 표시 확인
7. 최대 12장 제한 확인
8. 12장 도달 시 추가 버튼 미표시 확인
9. 추가 생성 후 결과함에 그대로 남는지 확인
10. 원본 파일이 없을 때 기존 결과 이미지 참고 로직 확인
11. 이미지 데이터 과대 오류 방지를 위한 참고 이미지 압축 로직 확인
12. 문서 변경 시 사용설명서, 작업지시서, 작업확인서 버전 동기화

## 6. V1 전환 시 반드시 재검토할 항목

V1에서 Supabase 저장을 도입하면 다음 기능을 다시 연결해야 한다.

1. 결과 저장
2. 최근 제작 작업 목록
3. 제목 수정
4. 결과 이미지 다시 열기
5. 페이지 추가 생성
6. 원본 업로드 자료 보관
7. ZIP 다운로드
8. 개별 이미지 편집
9. AI 멘트 생성
10. 사용자별 권한과 저장소 분리

V1 작업 시 V0.1.2 기능이 퇴행하지 않는지 반드시 확인한다.

## 7. V0 메뉴 운영 결정

V0.1.2에서는 아래 정책을 따른다.

1. `나머지 섹션 만들기`
   - 화면에서는 숨김 처리한다.
   - 내부 함수는 당분간 보존할 수 있다.
   - V1 완성 후 재도입 여부를 검토한다.

2. `페이지 추가 생성`
   - 유지한다.
   - 결과 마지막 페이지 뒤에 1장, 2장, 3장 단위로 추가한다.
   - 최대 12장까지만 생성한다.

3. `생성 페이지`
   - V0에서는 `1장(히어로)`, `4장(심플)`, `8장(상세)` 구조를 유지한다.
   - `6장`, `10장`, `12장` 초기 생성 옵션은 V1 완성 후 재검토한다.

V1 작업이 완료되면 마스터에게 위 3개 메뉴 정책을 반드시 다시 보고하고 컨펌을 받아야 한다.
