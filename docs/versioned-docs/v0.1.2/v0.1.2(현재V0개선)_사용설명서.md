# v0.1.2 사용설명서

이 문서는 V1 클라우드 저장으로 넘어가기 전, 현재 V0 서비스에 누적된 기능을 기준으로 작성한 사용 설명서이다.

## 1. 버전 범위

v0.1.2는 다음 기준의 서비스 상태를 설명한다.

- 브라우저 저장 기반
- 로컬 ZIP 다운로드
- OpenAI Image 2.0 / Google Nano Banana 2 생성
- 개별 이미지 편집
- AI 멘트 생성
- 1장, 4장, 8장 생성 옵션
- 9:16, 1:1, 4:5 출력 비율
- 최대 12장까지 페이지 추가 생성
- 최근 제작 작업 제목 수정
- 모델/인물 이미지 노출 규칙

## 2. 기본 사용 흐름

1. 사이트에 접속한다.
2. 메인 화면에서 `phoenix detail page 입장`을 누른다.
3. `API 키 설정`에서 OpenAI 또는 Google API 키를 입력한다.
4. `제작실`에서 기존 상세페이지 이미지 또는 PDF를 업로드한다.
5. 생성 페이지, 출력 비율, 사용처를 선택한다.
6. 제작 요청 메모를 입력한다.
7. `이미지 제작 시작`을 누른다.
8. 결과를 확인하고 필요하면 개별 편집 또는 페이지 추가 생성을 실행한다.
9. `결과 저장` 또는 `결과 다운로드`를 사용한다.

## 3. 생성 페이지

- `1장(히어로)`: 첫 화면 테스트, 제품/대상/혜택 빠른 확인
- `4장(심플)`: 짧은 상세페이지 초안, 빠른 검토용
- `8장(상세)`: 기본 상세페이지 구성

## 4. 출력 비율

- `9:16(상세)`: 모바일 상세페이지 본문에 적합
- `1:1(정사각)`: 썸네일, 카드뉴스, 보조 이미지에 적합
- `4:5(피드)`: 광고, SNS 피드 소재에 적합

결과 이미지는 선택한 비율에 맞춰 정리된다.

## 5. 결과 저장과 다운로드

`결과 저장`은 현재 브라우저 저장소에 작업을 저장한다.

주의:

- 같은 브라우저에서는 저장된 작업을 다시 열 수 있다.
- 다른 PC나 다른 브라우저에서는 저장 내용이 보이지 않을 수 있다.
- 브라우저 데이터를 삭제하면 저장된 작업이 사라질 수 있다.

`결과 다운로드`는 생성된 이미지를 ZIP 파일 하나로 다운로드한다.

## 6. 페이지 추가 생성

결과 마지막 페이지 뒤에 `페이지 추가 생성` 카드가 표시된다.

가능한 버튼:

- `1장 추가하기`
- `2장 추가하기`
- `3장 추가하기`

최대 생성량은 12장이다.

현재 장수에 따라 가능한 버튼만 표시된다.

- 현재 8장: 1장, 2장, 3장 추가 가능
- 현재 10장: 1장, 2장 추가 가능
- 현재 11장: 1장 추가 가능
- 현재 12장: 추가 불가

원본 이미지나 PDF가 현재 작업 화면에 남아 있으면 추가 생성 정확도가 높다.

원본 자료가 없을 때는 기존 결과 이미지를 참고 자료로 사용한다. 이때 요청 데이터가 너무 커지지 않도록 최근 결과 이미지 일부를 압축해 참고한다.

## 7. 개별 이미지 편집

각 결과 이미지 카드에서 부분 편집을 할 수 있다.

사용 방법:

1. `부분 편집 메모`에 수정 요청을 입력한다.
2. 필요하면 빠른 수정 버튼을 누른다.
3. `이 이미지 편집`을 누른다.

빠른 수정 버튼:

- 카피 강화
- 디자인 강화
- CTA 강화
- 모바일 첫화면
- 리뷰/UGC 신뢰
- 불안 제거
- 중복 레이아웃 줄이기
- 안전 표현

## 8. AI 멘트 생성

빠른 수정 버튼 옆의 `AI 멘트 생성`을 누르면 해당 목적에 맞는 편집 요청 문구를 자동으로 생성한다.

이 기능은 `API 키 설정`에 입력한 OpenAI API 키를 사용한다.

## 9. 최근 제작 작업

홈보드의 `최근 제작 작업`에서 저장된 작업을 다시 열 수 있다.

가능한 작업:

- 제목 클릭: 결과 열기
- 연필 버튼: 제목 수정
- 체크 버튼: 제목 저장
- X 버튼: 제목 수정 취소
- 휴지통 버튼: 작업 삭제

## 10. 모델/인물 이미지 규칙

모델이나 인물 이미지가 너무 반복 노출되지 않도록 생성 규칙이 적용된다.

- 홀수 페이지: 모델/인물 1명 이하 허용
- 짝수 페이지: 제품컷, 아이콘, 표, 카드, FAQ, 근거 자료 중심

## 11. V1 전환 전 주의

v0.1.2는 브라우저 저장 기반이다.

V1에서는 Phoenix Supabase Storage와 Database를 도입해 다음 기능이 개선될 예정이다.

- 저장된 프로젝트 재열기 안정화
- 원본 이미지 보관
- 결과 이미지 URL 저장
- 로그인/유료 유저별 프로젝트 분리
- 추가 생성과 개별 편집의 클라우드 기반 안정화
