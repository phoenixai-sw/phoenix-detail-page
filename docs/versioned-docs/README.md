# phoenix detail page 버전 문서 관리

이 폴더는 phoenix detail page의 실제 운영 문서를 버전별로 관리한다.

`upgrade-roadmap`은 V0, V1, V1.5, V2, V3처럼 큰 제품 진화 단계를 설명하고, 이 폴더는 각 단계 안에서 실제로 배포/운영되는 세부 버전 문서를 관리한다.

## 버전 체계

| 버전 | 의미 | 저장/운영 기준 |
| --- | --- | --- |
| V0.x | 클라우드 저장 이전의 로컬/브라우저 기반 서비스 개선 | 브라우저 저장, 로컬 다운로드, 현재 작업 흐름 중심 |
| V1.x | Phoenix Cloud 저장 도입 이후 개선 | Google 로그인, Supabase Storage + Database, 슈퍼 관리자/일반 유저 |
| V1.5.x | 결제/크레딧 도입 이후 개선 | Toss 결제, 무료 크레딧, 유료 크레딧, 사용량 차감 |
| V2.x | Design bot 텔레그램 자동화 이후 개선 | Design bot이 웹사이트를 조작하고 결과를 텔레그램으로 전달 |
| V3.x | Genesis bot 지휘 자동화 이후 개선 | Genesis bot이 Design bot을 지휘하는 비서실장형 자동화 |

## 현재 기준 버전

현재 기준 문서는 `v0.1.2`이다.

`v0.1.2`는 V1으로 넘어가기 전, V0 서비스에 누적된 최근 기능 개선을 반영한 운영 문서 버전이다.

## 문서 관리 원칙

1. V1으로 가기 전까지 추가되는 기능은 모두 V0.x 문서에 누적 반영한다.
2. 기능이 바뀌면 사용설명서, 코덱스용 작업지시서, 유저용 작업확인서를 함께 확인한다.
3. 저장 구조가 바뀌는 작업은 반드시 V1.x 문서에서 별도 버전으로 관리한다.
4. 결제와 크레딧 구조가 바뀌는 작업은 V1.5.x 문서에서 관리한다.
5. 봇 자동화가 붙는 작업은 V2.x 또는 V3.x 문서에서 관리한다.
6. 기존 기능이 새 저장 구조와 충돌하지 않는지 작업지시서에 확인 항목으로 남긴다.
7. 루트의 `docs/phoenix-detail-page-사용설명서.md`는 현재 사용자를 위한 최신 설명서이고, 이 폴더는 버전 기록용 기준 문서이다.
