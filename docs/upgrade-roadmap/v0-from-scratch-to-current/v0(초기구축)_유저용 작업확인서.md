# V0(초기구축) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 입문자 유저가 Phoenix detail page 기본 사이트가 제대로 구축되었는지 화면을 보며 확인하는 문서다.

V0의 최종 상태는 아래와 같다.

```text
로그인 없이 사용 가능
유저가 자기 API 키 입력
상세페이지 이미지 생성
개별 이미지 편집
AI 멘트 생성
브라우저 저장
로컬 다운로드
```

## 2. 준비물

1. Chrome 또는 Edge 브라우저
2. OpenAI API 키
3. Google API 키
4. 테스트용 상세페이지 이미지 또는 PDF
5. 로컬 실행 시 Node.js

둘 중 하나의 API 키만 있어도 테스트는 가능하다.

```text
OpenAI만 있으면 OpenAI Image 2.0 테스트
Google만 있으면 Nano Banana 2 테스트
둘 다 있으면 둘 다 테스트
```

## 3. 사이트 접속 확인

### 3.1 로컬 서버 실행

프로젝트 폴더에서 아래 명령을 실행한다.

```bash
npm run dev:local
```

### 3.2 메인 페이지 접속

브라우저 주소창에 아래 주소를 입력한다.

```text
http://localhost:3002
```

확인할 것:

1. Phoenix 로고가 보이는가
2. 메인 이미지가 위아래 잘리지 않는가
3. `phoenix detail page 입장` 버튼이 보이는가
4. 하단 Phoenix AI 푸터가 보이는가

### 3.3 작업 화면 입장

`phoenix detail page 입장` 버튼을 누른다.

성공 기준:

```text
/studio 작업 화면으로 이동한다.
```

## 4. 왼쪽 메뉴 확인

작업 화면 왼쪽에서 아래 메뉴를 확인한다.

1. 홈보드
2. 새 작업
3. 제작 결과

### 4.1 홈보드

확인할 것:

1. 최근 작업 목록 영역이 보이는가
2. 저장된 작업이 없을 경우 빈 상태 안내가 보이는가
3. 맞춤형 Data 설정 버튼이 보이는가
4. API 키 설정 버튼이 보이는가

### 4.2 새 작업

확인할 것:

1. 이미지 또는 PDF 업로드 영역이 보이는가
2. 제작 엔진 선택이 보이는가
3. 판매 채널 선택이 보이는가
4. 결과 장수 선택이 보이는가
5. 출력 비율 선택이 보이는가
6. 요청 문구 입력창이 보이는가
7. 생성 버튼이 보이는가

### 4.3 제작 결과

생성 전에는 결과가 없을 수 있다.
생성 후에는 결과 카드가 표시되어야 한다.

## 5. 설정 박스 확인

왼쪽 또는 상단의 설정 박스에서 아래 상태를 확인한다.

1. OpenAI Image 2.0
2. Google Nano Banana 2
3. 맞춤형 Data 설정

키가 없으면 `미설정`으로 표시된다.
키를 입력하면 해당 항목이 `설정됨`으로 바뀌어야 한다.

## 6. API 키 설정

### 6.1 API 키 설정 열기

`API 키 설정` 버튼을 누른다.

확인할 입력칸:

1. OpenAI Image 2.0 API 키
2. Google Nano Banana 2 API 키

### 6.2 OpenAI 키 입력

OpenAI API 키를 입력한다.

예시 형태:

```text
sk-...
```

### 6.3 Google 키 입력

Google API 키를 입력한다.

예시 형태:

```text
AIza...
```

### 6.4 저장

`저장` 버튼을 누른다.

성공 기준:

```text
API 키 설정을 저장했습니다.
```

같은 알림이 표시된다.

### 6.5 입력값 초기화

키를 지우고 싶으면 `입력값 초기화`를 누른다.

## 7. 맞춤형 Data 설정 확인

`맞춤형 Data 설정` 버튼을 누른다.

확인할 것:

1. PDF/TXT/MD 파일 등록 영역이 보이는가
2. 등록된 Data 목록이 보이는가
3. 등록된 파일이 없으면 안내 문구가 보이는가

V0에서는 이 기능이 기본 구조로만 있어도 된다.
서버 RAG 저장소까지 완전 운영하는 것은 별도 설정이 필요하다.

## 8. 새 작업 생성

### 8.1 새 작업 화면으로 이동

왼쪽 메뉴에서 `새 작업`을 누른다.

### 8.2 파일 업로드

업로드 영역을 클릭하거나 파일을 끌어다 놓는다.

지원 파일:

```text
이미지
PDF
```

성공 기준:

```text
업로드된 파일 이름 또는 등록 개수가 표시된다.
```

### 8.3 제작 엔진 선택

아래 중 하나를 선택한다.

1. OpenAI Image 2.0
2. Google Nano Banana 2

선택한 엔진에 맞는 API 키가 있어야 한다.

### 8.4 판매 채널 선택

판매 채널 옵션을 선택한다.

예시:

```text
스마트스토어
쿠팡
맞춤형 웹/앱
```

### 8.5 결과 장수 선택

처음 테스트할 때는 1장을 권장한다.

```text
히어로 1장
```

### 8.6 출력 비율 선택

상세페이지 세로형은 보통 아래 비율을 선택한다.

```text
9:16
```

### 8.7 요청 문구 입력

요청 문구 입력창에 원하는 방향을 적는다.

예시:

```text
첫 화면에서 제품의 차별점이 바로 보이게 하고, 구매 불안을 줄이는 근거 섹션을 강화해주세요.
```

### 8.8 생성 시작

생성 버튼을 누른다.

성공 기준:

1. 진행 패널이 표시된다.
2. 생성 중 문구가 표시된다.
3. 완료 후 결과 화면으로 이동한다.

## 9. 결과 화면 확인

결과 화면에서 아래 버튼을 확인한다.

1. 결과 저장
2. 히어로 다시 생성
3. 결과 다운로드

### 9.1 결과 카드 확인

각 결과 카드에서 확인할 것:

1. 이미지가 보이는가
2. 섹션명이 보이는가
3. 섹션 목적이 보이는가
4. 원본 참조가 보이는가
5. 이미지 다운로드 버튼이 보이는가
6. 부분 편집 메모 입력창이 보이는가

### 9.2 결과 다운로드

`결과 다운로드`를 누른다.

성공 기준:

```text
이미지 파일이 PC에 다운로드된다.
```

### 9.3 결과 저장

`결과 저장`을 누른다.

성공 기준:

```text
결과를 저장했습니다. 홈보드에서 다시 열 수 있습니다.
```

또는 비슷한 저장 완료 메시지가 보인다.

### 9.4 홈보드에서 다시 열기

1. 왼쪽 메뉴에서 `홈보드`를 누른다.
2. 최근 작업 목록에서 방금 저장한 작업을 찾는다.
3. 작업을 클릭한다.

성공 기준:

```text
저장했던 결과 화면이 다시 열린다.
```

## 10. 섹션별 빠른 편집 확인

결과 카드의 `부분 편집 메모` 아래에 아래 메뉴들이 보여야 한다.

1. 카피 강화
2. 디자인 강화
3. CTA 강화
4. 모바일 첫화면
5. 리뷰/UGC 신뢰
6. 불안 제거
7. 중복 레이아웃 줄이기
8. 안전 표현

각 메뉴를 누르면 기본 문구가 입력창에 들어간다.

확인 방법:

1. `카피 강화`를 누른다.
2. 입력창에 문구가 들어가는지 확인한다.
3. `디자인 강화`를 누른다.
4. 기존 문구 아래에 새 문구가 추가되는지 확인한다.

## 11. AI 멘트 생성 확인

각 빠른 편집 메뉴 옆의 `AI 멘트 생성` 버튼을 누른다.

성공 기준:

1. 버튼에 로딩 표시가 나온다.
2. OpenAI API 키를 사용해 문구를 생성한다.
3. 생성된 문구가 부분 편집 메모 입력창에 추가된다.

실패 시 확인:

1. OpenAI API 키가 입력되어 있는가
2. API 키가 만료되지 않았는가
3. 계정에 모델 사용 권한이 있는가

## 12. 개별 이미지 편집 확인

### 12.1 편집 문구 입력

부분 편집 메모에 직접 입력하거나 빠른 편집 메뉴를 누른다.

예시:

```text
헤드라인을 더 짧고 명확하게 바꾸고, CTA 버튼을 조금 더 잘 보이게 정리해주세요.
```

### 12.2 편집 엔진 선택

아래 중 하나를 선택한다.

1. OpenAI Image 2.0
2. Nano Banana 2

### 12.3 이 이미지 편집

`이 이미지 편집` 버튼을 누른다.

성공 기준:

1. 편집 진행 상태가 표시된다.
2. 완료 후 수정 이미지가 카드에 반영된다.
3. 이미지 히스토리에서 원본과 수정본을 넘겨볼 수 있다.

### 12.4 요청 크기 오류 확인

아래 오류가 반복되면 안 된다.

```text
이미지 데이터가 너무 커서 부분 편집 요청을 보낼 수 없습니다.
```

V0 안정화 기준은 FormData 방식으로 개별 편집 요청을 보내는 것이다.

## 13. 빌드 확인

프로젝트 폴더에서 아래 명령을 실행한다.

```bash
npm run build
```

성공 기준:

```text
Compiled successfully
```

또는 빌드 성공 메시지가 표시된다.

## 14. V0에서 확인하지 않는 것

아래 기능은 V0 범위가 아니다.

1. 로그인
2. 유료 유저 구분
3. Phoenix Supabase Storage 저장
4. Phoenix Supabase Database 저장
5. 고급 유저 자기 Supabase 연결
6. 다른 PC에서 프로젝트 이어 열기

위 기능은 V1 또는 V2에서 확인한다.
