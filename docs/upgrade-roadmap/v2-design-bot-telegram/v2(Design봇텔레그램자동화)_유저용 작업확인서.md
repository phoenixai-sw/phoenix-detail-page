# V2(Design봇텔레그램자동화) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 Design bot이 텔레그램 명령을 받아 Phoenix detail page 웹사이트를 조작하고, 상세페이지 생성 결과물을 텔레그램으로 돌려주는지 확인하는 문서다.

## 2. 준비물

1. 실행 가능한 Design bot
2. Design bot 텔레그램 봇
3. Phoenix detail page 사이트 주소
4. 로그인 계정
5. OpenAI 또는 Google API 키
6. 테스트용 이미지 또는 PDF
7. 결과를 받을 텔레그램 채팅방

## 3. Design bot 실행 확인

1. OpenClaw 또는 PM2에서 Design bot 상태를 확인한다.
2. Design bot이 online 상태인지 확인한다.
3. 텔레그램에서 Design bot에게 `/start`를 보낸다.

성공 기준:

```text
Design bot이 응답한다.
```

## 4. 상세페이지 생성 명령 시작

텔레그램에서 아래 명령을 보낸다.

```text
/detail_create
```

성공 기준:

1. 봇이 상품명 입력을 요청한다.
2. 판매 채널 선택을 요청한다.
3. 생성 페이지 수 선택을 요청한다.
4. 출력 비율 선택을 요청한다.
5. 참고 파일 업로드를 요청한다.

## 5. 파일 업로드 확인

텔레그램에 테스트용 이미지 또는 PDF를 업로드한다.

성공 기준:

```text
파일을 받았습니다.
```

또는 비슷한 메시지가 표시된다.

## 6. 생성 조건 입력

예시:

```text
상품명: 피닉스 테스트 상품
생성 페이지: 1장
출력 비율: 9:16
판매 채널: 맞춤형 웹/앱
요청 문구: 모바일 첫 화면에서 혜택이 잘 보이게
```

성공 기준:

1. 봇이 입력 내용을 요약한다.
2. 생성 시작 여부를 확인한다.

## 7. 웹사이트 자동 조작 확인

Design bot이 Phoenix detail page를 열고 작업을 시작한다.

확인할 것:

1. 사이트 접속
2. 로그인 또는 API 키 설정
3. 파일 업로드
4. 프롬프트 입력
5. 생성 버튼 클릭
6. 결과 대기

성공 기준:

```text
현재 상태가 텔레그램에 중간 보고된다.
```

## 8. 결과 수신 확인

생성이 끝나면 텔레그램에서 결과를 확인한다.

성공 기준:

1. ZIP 파일이 전달된다.
2. 대표 이미지가 함께 전달된다.
3. 프로젝트 링크가 있으면 함께 전달된다.
4. 완료 메시지가 표시된다.

## 9. 실패 상황 확인

일부러 실패 상황을 만든다.

예시:

1. 잘못된 API 키
2. 너무 큰 파일
3. 사이트 로그아웃
4. 생성 타임아웃

성공 기준:

```text
봇이 실패 원인을 설명하고 재시도 방법을 안내한다.
```

## 10. 동시 작업 확인

가능하면 서로 다른 텔레그램 유저 또는 같은 유저가 2개 작업을 순서대로 요청한다.

성공 기준:

1. 작업별 job_id가 다르다.
2. 파일이 섞이지 않는다.
3. 결과가 각각 올바른 채팅방으로 전달된다.

## 11. V2 성공 기준

1. 텔레그램 명령 수신
2. 파일 업로드 수신
3. 웹사이트 자동 접속
4. 상세페이지 생성 실행
5. ZIP 결과 다운로드
6. 텔레그램 결과 전달
7. 실패 시 원인 안내
8. 작업 파일 격리
