# V1(피닉스클라우드저장) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 로그인 유저 또는 유료 유저가 Phoenix cloud 저장 기능을 사용할 수 있는지 확인하는 문서다.

V1의 핵심은 아래와 같다.

```text
Google 로그인
클라우드 저장
다른 브라우저에서 다시 열기
Supabase Storage 이미지 확인
Supabase Database 프로젝트 정보 확인
슈퍼 관리자와 일반 유저 권한 구분
```

## 2. 준비물

1. Phoenix detail page 사이트 주소
2. 로그인 가능한 계정
3. 클라우드 저장 권한
4. OpenAI 또는 Google API 키
5. 테스트용 상세페이지 이미지 또는 PDF
6. Supabase Dashboard 접근 권한
7. 슈퍼 관리자 계정 1개
8. 일반 유저 계정 1개

## 3. 사이트 접속

브라우저에서 Phoenix detail page 사이트에 접속한다.

확인할 것:

1. 메인 페이지가 열리는가
2. `phoenix detail page 입장` 버튼이 보이는가
3. 버튼 클릭 시 `/studio`로 이동하는가

## 4. 로그인 메뉴 확인

작업 화면에서 로그인 관련 메뉴를 찾는다.

예상 메뉴:

1. 로그인
2. 로그아웃
3. 내 클라우드
4. 클라우드 저장

비로그인 상태에서 확인:

```text
로그인 버튼이 보여야 한다.
클라우드 저장 버튼은 숨겨져 있거나 비활성화되어야 한다.
```

## 5. 로그인하기

### 5.1 로그인 버튼 클릭

`로그인` 버튼을 누른다.

### 5.2 Google 로그인 선택

`Google로 로그인` 버튼을 누른다.

### 5.3 로그인 완료 확인

성공 기준:

```text
유저 이메일 또는 유저명이 화면에 표시된다.
로그아웃 버튼이 표시된다.
```

실패 시 확인:

1. 이메일 주소가 맞는가
2. 로그인 링크가 만료되지 않았는가
3. Google 계정 선택을 완료했는가
4. 클라우드 저장 허용 계정인가

## 5.4 슈퍼 관리자 로그인 확인

슈퍼 관리자 계정으로 로그인한다.

확인할 메뉴:

```text
관리자 모드
```

성공 기준:

1. 관리자 모드 버튼이 보인다.
2. 관리자 화면에 들어갈 수 있다.
3. 전체 유저 목록이 보인다.
4. 전체 프로젝트 목록이 보인다.

실패 시 확인:

1. Supabase `user_roles`에 해당 계정이 `super_admin`으로 등록됐는가
2. 배포 환경변수에 슈퍼 관리자 허용 이메일이 있다면 이메일이 정확한가
3. 로그아웃 후 다시 로그인했는가

## 5.5 일반 유저 로그인 확인

일반 유저 계정으로 로그인한다.

성공 기준:

1. `내 클라우드` 메뉴는 보인다.
2. `관리자 모드` 메뉴는 보이면 안 된다.
3. 다른 유저 프로젝트가 보이면 안 된다.

## 6. API 키 설정

`API 키 설정`을 연다.

입력할 것:

1. OpenAI API 키
2. Google API 키

둘 중 하나만 있어도 테스트 가능하다.

저장 후 확인:

```text
설정 박스에서 해당 API가 설정됨으로 표시된다.
```

## 7. 상세페이지 생성

### 7.1 새 작업 이동

`새 작업` 메뉴를 누른다.

### 7.2 파일 업로드

상세페이지 이미지 또는 PDF를 업로드한다.

### 7.3 제작 조건 선택

확인할 메뉴:

1. 제작 엔진
2. 판매 채널
3. 결과 장수
4. 출력 비율
5. 요청 문구

처음 테스트는 1장 생성으로 진행한다.

### 7.4 생성 실행

생성 버튼을 누른다.

성공 기준:

```text
결과 이미지가 생성된다.
```

## 8. 클라우드 저장

### 8.1 클라우드 저장 버튼 확인

결과 화면에서 아래 버튼을 확인한다.

```text
클라우드 저장
```

또는 `결과 저장` 버튼 안에 클라우드 저장 옵션이 있을 수 있다.

### 8.2 클라우드 저장 실행

`클라우드 저장`을 누른다.

성공 기준:

```text
클라우드에 저장했습니다.
```

또는 비슷한 저장 완료 메시지가 표시된다.

### 8.3 저장 실패 시 확인

1. 로그인이 되어 있는가
2. 클라우드 저장 권한이 있는가
3. Supabase 환경변수가 배포 사이트에 들어가 있는가
4. Storage 버킷이 존재하는가
5. 저장 용량 제한을 넘지 않았는가

## 9. 내 클라우드 목록 확인

`내 클라우드` 메뉴를 누른다.

확인할 것:

1. 저장한 프로젝트 제목이 보이는가
2. 생성일 또는 저장일이 보이는가
3. 사용한 모델이 보이는가
4. 이미지 개수가 보이는가

프로젝트를 클릭한다.

성공 기준:

```text
저장된 상세페이지 결과가 다시 열린다.
```

## 10. 다른 브라우저에서 확인

1. 다른 브라우저를 연다.
2. 같은 사이트에 접속한다.
3. 같은 계정으로 로그인한다.
4. `내 클라우드` 메뉴를 연다.
5. 저장한 프로젝트가 보이는지 확인한다.

성공 기준:

```text
다른 브라우저에서도 저장 프로젝트가 보인다.
```

## 11. Supabase Storage 확인

Supabase Dashboard에 접속한다.

이동:

```text
Storage
```

확인할 버킷:

```text
detail-page-images
```

확인할 것:

1. 유저별 폴더가 생겼는가
2. 프로젝트별 폴더가 생겼는가
3. 섹션 이미지 파일이 저장됐는가
4. 수정 이미지 파일이 저장됐는가

예상 경로:

```text
users/{userId}/projects/{projectId}/...
```

## 12. Supabase Database 확인

Supabase Dashboard에서 아래로 이동한다.

```text
Table Editor
```

확인할 테이블 예시:

1. `profiles`
2. `cloud_projects`
3. `cloud_project_sections`
4. `cloud_project_images`
5. `user_entitlements`

확인할 것:

1. 로그인 유저 ID가 저장됐는가
2. 프로젝트 제목이 저장됐는가
3. 이미지 URL 또는 storage path가 저장됐는가
4. 생성일과 수정일이 저장됐는가

## 13. 프로젝트 삭제 확인

`내 클라우드`에서 프로젝트 삭제를 실행한다.

성공 기준:

1. 화면 목록에서 사라진다.
2. Supabase Database row가 삭제된다.
3. Supabase Storage 이미지도 삭제된다.

## 14. 권한 확인

테스트 가능하다면 다른 계정으로 로그인한다.

확인할 것:

```text
다른 계정의 프로젝트가 보이면 안 된다.
```

## 14.1 슈퍼 관리자 권한 확인

슈퍼 관리자 계정으로 다시 로그인한다.

확인할 것:

1. 일반 유저가 저장한 프로젝트가 보인다.
2. 프로젝트 상세를 열 수 있다.
3. 유저별 프로젝트 개수 또는 저장량을 확인할 수 있다.
4. 일반 유저의 클라우드 저장 권한을 변경할 수 있다.

주의:

```text
슈퍼 관리자는 운영 확인용으로 전체 데이터를 볼 수 있지만, 일반 유저 화면에서는 절대 다른 유저 데이터가 보이면 안 된다.
```

## 14.2 일반 유저 권한 차단 확인

일반 유저 계정으로 아래를 확인한다.

1. 관리자 화면 URL을 직접 입력해 본다.
2. 다른 유저 프로젝트 ID가 포함된 URL을 직접 입력해 본다.
3. 다른 유저 Storage 이미지 URL 접근을 시도한다.

성공 기준:

```text
접근이 거부되거나 내 데이터만 보여야 한다.
```

## 15. V1 성공 기준

1. 로그인 가능
2. 로그아웃 가능
3. 클라우드 저장 가능
4. 내 클라우드 목록에서 다시 열기 가능
5. 다른 브라우저에서 다시 열기 가능
6. Supabase Storage에 이미지 저장
7. Supabase Database에 프로젝트 정보 저장
8. 프로젝트 삭제 시 DB와 Storage 정리
9. 다른 유저 프로젝트 접근 차단
10. 슈퍼 관리자 전체 조회 가능
11. 일반 유저 관리자 모드 차단
