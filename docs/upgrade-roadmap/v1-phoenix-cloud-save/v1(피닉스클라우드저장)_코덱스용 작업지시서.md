# V1(피닉스클라우드저장) 코덱스용 작업지시서

## 1. 작업 목표

V1은 로그인 유저 또는 유료 유저가 Phoenix가 운영하는 Supabase에 프로젝트와 이미지를 저장할 수 있게 만드는 단계다.

구조는 아래와 같다.

```text
Supabase Auth = 로그인
Supabase Storage = 이미지 파일 저장
Supabase Database = 프로젝트 정보, 이미지 URL, 유저 정보 저장
```

V1은 모든 공개 유저의 이미지를 자동 저장하는 기능이 아니다.
클라우드 저장은 로그인/유료 유저에게만 제공한다.

## 2. V1 구현 전 확인할 정책

구현 전에 마스터에게 아래 내용을 확인한다.

1. 로그인 방식
   - 이메일 로그인
   - Google 로그인
   - 둘 다 지원

2. 유료 유저 구분 방식
   - 일단 관리자 수동 승인
   - 결제 연동은 추후
   - 특정 이메일만 허용

3. 저장 제한
   - 유저당 프로젝트 개수
   - 유저당 이미지 용량
   - 무료/유료 구분 여부

4. 삭제 정책
   - 유저가 프로젝트 삭제 시 Storage 이미지도 삭제할지
   - 일정 기간 후 자동 삭제할지

## 3. 필요한 Supabase 구성

### 3.1 Supabase Auth

필요 기능:

1. 로그인
2. 로그아웃
3. 현재 로그인 유저 확인
4. 유저 ID 확보

### 3.2 Supabase Storage

권장 버킷:

```text
detail-page-images
```

권장 경로:

```text
users/{userId}/projects/{projectId}/sections/{sectionId}.png
users/{userId}/projects/{projectId}/revisions/{sectionId}-{revisionId}.png
```

### 3.3 Supabase Database

권장 테이블:

```text
profiles
cloud_projects
cloud_project_sections
cloud_project_images
user_entitlements
```

### 3.4 RLS 정책

필수 조건:

1. 유저는 자기 프로젝트만 볼 수 있다.
2. 유저는 자기 이미지만 업로드할 수 있다.
3. 유저는 자기 이미지만 삭제할 수 있다.
4. 관리자는 전체 조회 가능하다.

## 4. 환경변수

V1에서 필요한 환경변수 예시:

```env
NEXT_PUBLIC_SUPABASE_URL=""
NEXT_PUBLIC_SUPABASE_ANON_KEY=""
SUPABASE_SERVICE_ROLE_KEY=""
PHOENIX_CLOUD_ENABLED="true"
```

주의:

```text
SUPABASE_SERVICE_ROLE_KEY는 브라우저로 보내지 않는다.
```

## 5. 코드 구현 범위

### 5.1 Supabase 클라이언트

추가 파일 예시:

```text
src/lib/supabase/client.ts
src/lib/supabase/server.ts
src/lib/cloud-storage.ts
```

구현:

1. 브라우저용 Supabase client
2. 서버용 Supabase admin client
3. Storage 업로드 함수
4. Storage 삭제 함수
5. Database 저장 함수

### 5.2 로그인 UI

수정 대상:

```text
src/components/redesign-wizard.tsx
```

추가 메뉴:

1. 로그인
2. 로그아웃
3. 내 클라우드
4. 클라우드 저장

표시 조건:

```text
비로그인: 로그인 버튼 표시
로그인: 유저 이메일, 로그아웃, 클라우드 저장 표시
유료/승인 유저: 클라우드 저장 활성화
미승인 유저: 클라우드 저장 비활성화 또는 안내 표시
```

### 5.3 클라우드 저장 API

추가 API 예시:

```text
src/app/api/cloud/projects/route.ts
src/app/api/cloud/projects/[projectId]/route.ts
src/app/api/cloud/images/route.ts
```

기능:

1. 프로젝트 저장
2. 프로젝트 목록 조회
3. 프로젝트 상세 조회
4. 프로젝트 삭제
5. 이미지 Storage 업로드
6. 이미지 Storage 삭제

### 5.4 저장 흐름

기존 V0 저장:

```text
브라우저 IndexedDB 저장
```

V1 저장:

```text
1. 브라우저 IndexedDB 저장 유지
2. 로그인/유료 유저가 클라우드 저장 선택
3. 이미지 dataUrl을 Blob으로 변환
4. Supabase Storage에 업로드
5. Storage path 또는 public/signed URL 확보
6. Supabase Database에 프로젝트 메타데이터 저장
7. 내 클라우드 목록에서 다시 불러오기
```

## 6. V1에서 유지할 것

1. V0의 브라우저 저장 기능은 유지한다.
2. V0의 로컬 다운로드 기능은 유지한다.
3. 유저 API 키 입력 방식은 유지한다.
4. 비로그인 유저도 계속 생성/편집/다운로드 가능해야 한다.

## 7. V1에서 금지할 것

1. 비로그인 유저 이미지를 자동으로 Phoenix Supabase에 저장하지 않는다.
2. 유저 API 키를 Supabase Database에 저장하지 않는다.
3. service role key를 브라우저에 노출하지 않는다.
4. 저장 실패가 생성 실패로 이어지게 하지 않는다.

## 8. UI 성공 기준

1. 로그인 버튼이 보인다.
2. 로그인 성공 후 이메일 또는 유저명이 보인다.
3. 로그아웃이 된다.
4. 클라우드 저장 버튼이 보인다.
5. 승인 유저만 클라우드 저장을 할 수 있다.
6. 내 클라우드 목록에서 저장 프로젝트가 보인다.
7. 다른 브라우저에서 로그인해도 저장 프로젝트가 보인다.

## 9. 데이터 성공 기준

1. Supabase Storage에 이미지 파일이 저장된다.
2. Supabase Database에 프로젝트 정보가 저장된다.
3. 프로젝트 삭제 시 DB row가 삭제된다.
4. 프로젝트 삭제 시 Storage 파일도 삭제된다.
5. 유저 A는 유저 B 프로젝트를 볼 수 없다.

## 10. 검증 명령

```bash
npm run build
```

```text
/api/cloud/health
```

같은 경로로 상태를 확인한다.
