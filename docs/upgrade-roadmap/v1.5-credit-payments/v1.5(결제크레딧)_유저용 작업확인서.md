# V1.5(결제크레딧) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 결제와 크레딧 기능이 실제 화면에서 정상 작동하는지 확인하기 위한 체크리스트다.

V1.5 핵심 확인:

```text
무료 크레딧 지급
결제 후 크레딧 충전
생성 시 크레딧 차감
실패 시 크레딧 미차감
슈퍼 관리자 결제/크레딧 관리
```

## 2. 준비물

1. V1이 적용된 사이트 주소
2. 일반 유저 Google 계정
3. 슈퍼 관리자 Google 계정
4. Toss 테스트 결제 수단
5. OpenAI 또는 Google API 키
6. 테스트용 상세페이지 이미지 또는 PDF
7. Supabase Dashboard 접근 권한

## 3. 신규 유저 무료 크레딧 확인

1. 새 일반 유저 계정으로 Google 로그인한다.
2. 화면에서 `내 크레딧` 또는 크레딧 잔액 영역을 확인한다.

성공 기준:

```text
무료 크레딧이 표시된다.
```

확인할 것:

1. 무료 크레딧 수량이 정책과 맞는가
2. 새로고침해도 잔액이 유지되는가
3. 로그아웃 후 다시 로그인해도 잔액이 유지되는가

## 4. 결제 상품 확인

1. `크레딧 충전` 버튼을 누른다.
2. 결제 상품 목록을 확인한다.

확인할 것:

1. 상품명
2. 결제 금액
3. 지급 크레딧
4. 부가 혜택 또는 안내 문구

## 5. Toss 결제 테스트

1. 원하는 크레딧 상품을 선택한다.
2. Toss 결제창으로 이동한다.
3. 테스트 결제를 완료한다.
4. 사이트로 돌아온다.

성공 기준:

```text
결제가 완료되었고 크레딧이 충전되었습니다.
```

또는 비슷한 완료 메시지가 보여야 한다.

## 6. 결제 후 잔액 확인

결제 완료 후 `내 크레딧`을 확인한다.

성공 기준:

1. 결제 전 잔액보다 증가했다.
2. 지급 크레딧이 상품 정보와 일치한다.
3. 사용 내역에 `결제 충전` 기록이 남는다.

## 7. 생성 시 크레딧 차감 확인

1. 새 작업을 만든다.
2. API 키를 입력한다.
3. 1장 생성으로 테스트한다.
4. 생성 전 크레딧 잔액을 기록한다.
5. 생성 완료 후 잔액을 다시 확인한다.

성공 기준:

```text
생성 성공 후 정책에 맞는 크레딧이 차감된다.
```

## 8. 실패 시 크레딧 미차감 확인

가능하면 테스트 환경에서 일부러 실패 상황을 만든다.

예시:

1. 잘못된 API 키 입력
2. 잔액 부족 상태
3. 네트워크 실패 테스트

성공 기준:

```text
이미지 생성이 실패하면 크레딧이 차감되지 않는다.
```

## 9. 잔액 부족 확인

크레딧이 부족한 일반 유저로 로그인한다.

확인할 것:

1. 생성 버튼이 비활성화되는가
2. 충전 안내가 보이는가
3. 부족한 크레딧 수량이 안내되는가

## 10. 슈퍼 관리자 결제 관리 확인

슈퍼 관리자 계정으로 로그인한다.

확인할 메뉴:

```text
관리자 모드 > 결제/크레딧
```

확인할 것:

1. 전체 유저 크레딧 잔액
2. 결제 성공 내역
3. 결제 실패 내역
4. 유저별 사용량
5. 수동 크레딧 지급 버튼
6. 수동 크레딧 차감 버튼

## 11. 일반 유저 권한 확인

일반 유저 계정으로 로그인한다.

성공 기준:

1. 자기 크레딧만 보인다.
2. 다른 유저 결제 내역이 보이면 안 된다.
3. 관리자 결제 페이지에 접근하면 차단되어야 한다.

## 12. Supabase 확인

Supabase Dashboard에서 아래 테이블을 확인한다.

```text
credit_accounts
credit_ledger
payment_orders
usage_logs
```

성공 기준:

1. 결제 주문이 저장된다.
2. 결제 성공 시 원장 기록이 남는다.
3. 이미지 생성 성공 시 사용 기록이 남는다.
4. 잔액이 원장과 일치한다.

## 13. V1.5 성공 기준

1. 무료 크레딧 지급
2. Toss 결제 성공
3. 결제 금액별 크레딧 지급
4. 생성 성공 시 크레딧 차감
5. 실패 시 크레딧 미차감
6. 잔액 부족 시 생성 차단
7. 슈퍼 관리자 전체 결제/크레딧 확인
8. 일반 유저 자기 내역만 확인
