# V3(Genesis봇지휘자동화) 유저용 작업확인서

## 1. 이 문서의 목적

이 문서는 Genesis bot이 Design bot을 지휘해 Phoenix detail page 상세페이지를 생성하고, 결과를 텔레그램으로 받을 수 있는지 확인하는 문서다.

## 2. 준비물

1. 실행 가능한 Genesis bot
2. 실행 가능한 Design bot
3. Phoenix detail page 사이트 주소
4. 마스터 텔레그램 계정
5. 테스트용 이미지 또는 PDF
6. 사이트 로그인 계정
7. 생성에 필요한 API 키 또는 크레딧

## 3. Genesis bot 기본 응답 확인

텔레그램에서 Genesis bot에게 메시지를 보낸다.

예시:

```text
상세페이지 하나 만들어줘
```

성공 기준:

```text
Genesis bot이 상세페이지 생성에 필요한 정보를 질문한다.
```

## 4. 정보 수집 확인

Genesis bot이 아래 정보를 질문하는지 확인한다.

1. 상품명
2. 생성 페이지 수
3. 출력 비율
4. 판매 채널
5. 참고 이미지 또는 PDF
6. 원하는 분위기
7. 결과 전달 방식

성공 기준:

```text
부족한 정보를 한 번에 또는 순서대로 질문한다.
```

## 5. Design bot 지시 확인

필요 정보 입력이 끝나면 Genesis bot이 Design bot에게 작업을 넘긴다.

성공 기준:

1. Genesis bot이 작업 접수 메시지를 보낸다.
2. Design bot이 작업을 수락한다.
3. job_id가 생성된다.

예상 메시지:

```text
Design bot에 상세페이지 생성 작업을 전달했습니다.
```

## 6. 진행 상태 확인

작업 중 Genesis bot이 상태를 알려주는지 확인한다.

상태 예시:

1. 파일 확인 중
2. 사이트 접속 중
3. 상세페이지 생성 중
4. 결과 다운로드 중
5. 텔레그램 전송 중

성공 기준:

```text
오래 걸리는 작업에서 중간 상태가 표시된다.
```

## 7. 결과 수신 확인

생성이 끝나면 텔레그램 결과를 확인한다.

성공 기준:

1. ZIP 파일 또는 이미지 파일을 받는다.
2. 대표 미리보기를 받는다.
3. 프로젝트 링크가 있으면 받는다.
4. Genesis bot이 최종 완료 보고를 한다.

## 8. 실패 처리 확인

일부러 실패 상황을 만든다.

예시:

1. 참고 파일 미첨부
2. 크레딧 부족
3. Design bot 중지
4. 사이트 접속 실패

성공 기준:

```text
Genesis bot이 실패 원인을 알려주고 다음 행동을 제안한다.
```

## 9. 마스터 승인 확인

자동 실행 전 승인 정책이 켜져 있다면, Genesis bot이 실행 전 확인을 요청해야 한다.

확인 문구 예시:

```text
이 조건으로 Design bot에게 작업을 전달할까요?
```

성공 기준:

1. 승인 전에는 Design bot이 실행되지 않는다.
2. 승인 후에만 작업이 시작된다.

## 10. V3 성공 기준

1. Genesis bot이 자연어 요청을 이해한다.
2. 부족한 정보를 질문한다.
3. Design bot에게 구조화된 작업을 전달한다.
4. Design bot이 웹사이트를 조작한다.
5. 결과물이 텔레그램으로 전달된다.
6. Genesis bot이 최종 완료 보고를 한다.
7. 실패 시 원인과 다음 행동을 알려준다.
8. 마스터 승인 정책이 지켜진다.
