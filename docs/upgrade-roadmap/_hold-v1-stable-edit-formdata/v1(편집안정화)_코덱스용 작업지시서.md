# V1 코덱스용 작업지시서

## 버전 목표

개별 이미지 편집 요청을 JSON base64 방식에서 FormData 파일 업로드 방식으로 변경한다.
목표는 `이미지 데이터가 너무 커서 부분 편집 요청을 보낼 수 없습니다` 오류를 실질적으로 제거하는 것이다.

## 구현 범위

1. 프론트 편집 요청 변경
   - 대상 파일: `src/components/redesign-wizard.tsx`
   - 현재 JSON body에 `imageUrl` base64 문자열을 넣는 방식 제거
   - 압축된 이미지를 `Blob`으로 만들어 `FormData`에 넣는다.

2. 서버 편집 API 변경
   - 대상 파일: `src/app/api/edit-section/route.ts`
   - `request.json()` 대신 `request.formData()` 사용
   - `image` 파일 필드에서 Blob/File을 읽어 Buffer로 변환
   - `section`, `project`, `request`, `model`, `openaiKey`, `googleKey`는 문자열 필드로 받는다.

3. 데이터 최소화
   - 섹션 정보는 `id`, `name`, `purpose`, `source`, `prompt`만 전송한다.
   - `imageUrl`, `revisions`는 전송하지 않는다.

4. 압축 정책
   - 편집용 이미지는 브라우저에서 JPEG Blob으로 압축한다.
   - 목표 용량을 넘으면 단계적으로 해상도와 품질을 낮춘다.

## 금지 사항

1. `/api/generate` 최초 생성 흐름은 건드리지 않는다.
2. AI 멘트 생성 API는 건드리지 않는다.
3. Supabase Storage 또는 DB 기능을 이 버전에 섞지 않는다.
4. 유저 API 키 저장 방식을 바꾸지 않는다.

## 검증 기준

1. `npm run build` 성공
2. 개별 이미지 편집 성공
3. 1장 생성 후 S1 편집 성공
4. 여러 장 생성 후 각 섹션 편집 성공
5. 큰 이미지에서도 요청 크기 오류가 재현되지 않음

## 실패 시 점검

1. FormData 필드명이 프론트와 서버에서 같은지 확인
2. 서버에서 `image` 파일이 null인지 확인
3. OpenAI와 Google 편집 API에 전달하는 Buffer가 정상인지 확인
4. 413 오류가 계속 나면 압축 목표 용량을 더 낮춘다.
